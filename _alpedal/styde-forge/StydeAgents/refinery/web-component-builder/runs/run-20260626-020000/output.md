## 1. Architecture Overview

This blueprint explores five advanced Web Component patterns that bridge the gap between native custom elements and framework-level capabilities. Each pattern addresses a real production concern: server-renderability, style isolation + theming, DOM scoping for micro-frontends, form integration, and validation.

### Pattern Map

| Pattern | Key APIs | Browser Baseline |
|---|---|---|
| **Declarative Shadow DOM** | `<template shadowrootmode>` | Chrome 90+, Edge 90+, Safari 16.4+, Firefox 123+ |
| **Scoped Element Registries** | `ShadowRoot.createElement()`, `CustomElementRegistry` scoping | Chrome 130+ (new), polyfill available |
| **Form-Associated + Validity** | `ElementInternals`, `setFormValue()`, `setValidity()` | Chrome 77+, Edge 79+, Safari 16.4+, Firefox 98+ |
| **CSS Parts Theming** | `::part()`, `::theme()`, `exportparts` | Chrome 73+, Edge 79+, Safari 13.1+, Firefox 72+ |
| **SSR + DSD** | Declarative Shadow DOM in HTML stream | Streaming: Chrome/Edge. Safari/Firefox: polyfill via JS. |

## 2. Declarative Shadow DOM

Declarative Shadow DOM (DSD) allows you to define a shadow root **directly in HTML**—no JavaScript required. The shadow root is attached by the parser, eliminating the flash-of-unstyled-content (FOUC) and enabling server-side rendering.

### 2.1 The Core Syntax

```html
<!-- DSD in HTML: the parser attaches shadow root automatically -->
<my-card>
  <template shadowrootmode="open">
    <style>
      :host { display: block; border: 1px solid #ccc; border-radius: 8px; padding: 16px; }
      h2 { margin: 0 0 8px; color: var(--card-heading, #333); }
      slot[name="footer"] { display: block; border-top: 1px solid #eee; margin-top: 12px; padding-top: 12px; }
    </style>
    <h2><slot name="title">Default Title</slot></h2>
    <div class="body">
      <slot></slot>
    </div>
    <div class="footer">
      <slot name="footer"></slot>
    </div>
  </template>
  <span slot="title">My Card Title</span>
  <p>Card body content goes here.</p>
  <button slot="footer">Action</button>
</my-card>
```

### 2.2 Imperative Fallback (for browsers without DSD support)

```typescript
// dsd-polyfill.ts — Fallback for declarative Shadow DOM
export function polyfillDeclarativeShadowDOM(): void {
  // Check if the browser supports DSD natively
  const supportsDSD = 'shadowRootMode' in HTMLTemplateElement.prototype;
  if (supportsDSD) return;

  // Find all templates with shadowrootmode attribute
  document.querySelectorAll('template[shadowrootmode]').forEach((template) => {
    const mode = template.getAttribute('shadowrootmode') as ShadowRootMode;
    const host = template.parentElement;
    if (!host) return;

    const shadowRoot = host.attachShadow({ mode });
    shadowRoot.appendChild(template.content.cloneNode(true));
    template.remove();
  });
}
```

### 2.3 Hydration Strategy: Matching Existing Shadow Roots

When the browser already parsed DSD, the custom element's `constructor` runs and discovers an existing shadow root. Avoid double-attaching:

```typescript
// my-card.ts
class MyCard extends HTMLElement {
  #internals: ElementInternals | null = null;

  constructor() {
    super();

    // Check for an existing (server-rendered) shadow root
    const existingRoot = this.shadowRoot;
    if (existingRoot) {
      // DSD already created the shadow root — hydrate references
      this.#hydrate(existingRoot);
    } else {
      // No DSD — create shadow root programmatically (client-only)
      this.attachShadow({ mode: 'open' });
      this.shadowRoot!.innerHTML = this.#template();
    }
  }

  #template(): string {
    return `
      <style>
        :host { display: block; border: 1px solid #ccc; border-radius: 8px; padding: 16px; }
        h2 { margin: 0 0 8px; }
        slot[name="footer"] { display: block; border-top: 1px solid #eee; margin-top: 12px; padding-top: 12px; }
      </style>
      <h2><slot name="title">Default Title</slot></h2>
      <div class="body"><slot></slot></div>
      <div class="footer"><slot name="footer"></slot></div>
    `;
  }

  #hydrate(root: ShadowRoot): void {
    // Bind event listeners, grab references to internal elements
    const titleSlot = root.querySelector('slot[name="title"]');
    titleSlot?.addEventListener('slotchange', () => {
      console.log('Title slot content changed');
    });
  }
}

customElements.define('my-card', MyCard);
```

### 2.4 Streaming HTML with DSD (Out-of-Order Rendering)

Declarative Shadow DOM is **streaming-friendly**. The browser can render each `<template shadowrootmode>` block as soon as it arrives, even before the closing `</my-card>` tag:

```html
<!-- Server streams these chunks independently -->
<my-card>
  <template shadowrootmode="open">
    <style>:host{display:block;border:1px solid #ccc;padding:16px;}</style>
    <h2><slot name="title">Loading...</slot></h2>
    <div><slot></slot></div>
  </template>
  <!-- Content arrives later via another stream chunk -->
  <span slot="title">Streamed Title</span>
  <p>This content arrives after the shadow root is already attached!</p>
</my-card>
```

### 2.5 `adoptedStyleSheets` with DSD

Combine DSD with Constructed Stylesheets for zero-duplication styles across many instances:

```typescript
// shared-styles.ts
const cardStyles = new CSSStyleSheet();
cardStyles.replaceSync(`
  :host { display: block; border: 1px solid #ccc; border-radius: 8px; padding: 16px; }
  h2 { margin: 0 0 8px; }
`);

class StyledCard extends HTMLElement {
  constructor() {
    super();
    const root = this.shadowRoot ?? this.attachShadow({ mode: 'open' });
    // Adopt shared stylesheet (browser deduplicates automatically)
    root.adoptedStyleSheets = [cardStyles];

    if (!this.shadowRoot) {
      // Client-only render
      root.innerHTML = `<h2><slot name="title">Card</slot></h2><div><slot></slot></div>`;
    }
  }
}

customElements.define('styled-card', StyledCard);
```

> **SSR Note:** For DSD + `adoptedStyleSheets`, inline styles in `<style>` blocks inside the DSD template ensure the SSR output is self-contained. At hydration, `adoptedStyleSheets` replaces them.

## 3. Scoped Element Registries

Scoped Custom Element Registries allow a shadow root to have its **own** `CustomElementRegistry`, preventing naming collisions across different components or micro-frontends on the same page.

### 3.1 The Problem: Global Registry Collisions

```typescript
// Team A defines <my-button>
customElements.define('my-button', TeamAButton);

// Team B also defines <my-button> — CRASH!
customElements.define('my-button', TeamBButton); // ❌ DOMException: already defined
```

### 3.2 Scoped Registry Solution

```typescript
// scoped-registry.ts — using the Scoped Custom Element Registry API
class ShellComponent extends HTMLElement {
  constructor() {
    super();
    const root = this.attachShadow({
      mode: 'open',
      // Create a scoped registry for this shadow tree
      customElements: new CustomElementRegistry(),
    });

    // Register components ONLY within this shadow tree's scope
    root.customElements!.define('my-button', class extends HTMLElement {
      constructor() {
        super();
        this.attachShadow({ mode: 'open' });
        this.shadowRoot!.innerHTML = `<style>:host { display: inline-block; } button { padding: 8px 16px; }</style><button><slot></slot></button>`;
      }
    });

    root.customElements!.define('my-input', class extends HTMLElement {
      constructor() {
        super();
        this.attachShadow({ mode: 'open' });
        this.shadowRoot!.innerHTML = `<style>:host { display: block; } input { padding: 8px; border: 1px solid #ccc; border-radius: 4px; }</style><input type="text">`;
      }
    });

    root.innerHTML = `
      <my-button>Click Me (Scoped)</my-button>
      <my-input></my-input>
    `;
  }
}

customElements.define('shell-component', ShellComponent);
```

## 4. Form-Associated Custom Elements with Validity

Form-associated custom elements (FACE) participate in native HTML forms: their values are submitted, they fire `formdata` events, and they integrate with the Constraint Validation API.

### 4.1 Basic Form-Associated Element

```typescript
// my-text-input.ts
class MyTextInput extends HTMLElement {
  static formAssociated = true; // ← Key: opt into form participation

  #internals: ElementInternals;
  #input: HTMLInputElement | null = null;

  constructor() {
    super();
    this.#internals = this.attachInternals(); // ElementInternals is the bridge
    this.attachShadow({ mode: 'open' });
  }

  connectedCallback(): void {
    this.shadowRoot!.innerHTML = `
      <style>
        :host { display: inline-block; }
        :host([invalid]) input { border-color: #e11d48; }
        :host([invalid]) .error { display: block; }
        .error { display: none; color: #e11d48; font-size: 0.875rem; margin-top: 4px; }
        label { display: block; font-weight: 500; margin-bottom: 4px; }
        input { padding: 8px; border: 1px solid #ccc; border-radius: 4px; font-size: 1rem; width: 100%; box-sizing: border-box; }
        input:focus { outline: 2px solid #3b82f6; outline-offset: -1px; }
      </style>
      <label>${this.getAttribute('label') || ''}</label>
      <input type="${this.getAttribute('type') || 'text'}"
             placeholder="${this.getAttribute('placeholder') || ''}"
             value="${this.getAttribute('value') || ''}">
      <div class="error"></div>
    `;

    this.#input = this.shadowRoot!.querySelector('input')!;

    // Sync input value to form internals on every change
    this.#input.addEventListener('input', () => {
      const value = this.#input!.value;
      this.#internals.setFormValue(value);
      this.#validate(value);
    });

    // Initial validation
    this.#validate(this.#input.value);
  }

  // Reflect attribute changes to the input
  static get observedAttributes(): string[] {
    return ['value', 'disabled', 'placeholder', 'required', 'minlength', 'maxlength', 'pattern'];
  }

  attributeChangedCallback(name: string, _oldValue: string | null, newValue: string | null): void {
    if (!this.#input) return;
    switch (name) {
      case 'value':
        this.#input.value = newValue ?? '';
        this.#internals.setFormValue(newValue ?? '');
        break;
      case 'disabled':
        this.#input.disabled = newValue !== null;
        break;
      case 'placeholder':
        this.#input.placeholder = newValue ?? '';
        break;
    }
    this.#validate(this.#input.value);
  }

  // --- Validity State Machine ---
  #validate(value: string): void {
    const required = this.hasAttribute('required');
    const minLength = parseInt(this.getAttribute('minlength') || '0', 10);
    const maxLength = parseInt(this.getAttribute('maxlength') || '0', 10);
    const pattern = this.getAttribute('pattern');

    const errorEl = this.shadowRoot!.querySelector('.error') as HTMLElement;

    // Reset validity
    this.#internals.setValidity({});

    if (required && value.trim() === '') {
      this.#internals.setValidity(
        { valueMissing: true },
        'This field is required.',
        this.#input!
      );
      errorEl.textContent = 'This field is required.';
    } else if (minLength > 0 && value.length < minLength) {
      this.#internals.setValidity(
        { tooShort: true },
        `Minimum ${minLength} characters required.`,
        this.#input!
      );
      errorEl.textContent = `Minimum ${minLength} characters required.`;
    } else if (maxLength > 0 && value.length > maxLength) {
      this.#internals.setValidity(
        { tooLong: true },
        `Maximum ${maxLength} characters allowed.`,
        this.#input!
      );
      errorEl.textContent = `Maximum ${maxLength} characters allowed.`;
    } else if (pattern && !new RegExp(pattern).test(value)) {
      this.#internals.setValidity(
        { patternMismatch: true },
        'Please match the requested format.',
        this.#input!
      );
      errorEl.textContent = 'Please match the requested format.';
    } else {
      errorEl.textContent = '';
    }
  }

  // Expose validity state for programmatic access
  get validity(): ValidityState {
    return this.#internals.validity;
  }

  get validationMessage(): string {
    return this.#internals.validationMessage;
  }

  checkValidity(): boolean {
    return this.#internals.checkValidity();
  }

  reportValidity(): boolean {
    return this.#internals.reportValidity();
  }

  // Form reset support
  formResetCallback(): void {
    const defaultValue = this.getAttribute('value') || '';
    if (this.#input) {
      this.#input.value = defaultValue;
      this.#internals.setFormValue(defaultValue);
    }
    this.#validate(defaultValue);
  }

  // Form disabled state
  formDisabledCallback(disabled: boolean): void {
    if (this.#input) {
      this.#input.disabled = disabled;
    }
  }

  // Form state restore (e.g., browser back button)
  formStateRestoreCallback(state: string | File | FormData | null, _mode: 'restore' | 'autocomplete'): void {
    if (typeof state === 'string' && this.#input) {
      this.#input.value = state;
      this.#internals.setFormValue(state);
      this.#validate(state);
    }
  }
}

customElements.define('my-text-input', MyTextInput);
```

### 4.2 Advanced: Custom Validation Rules with `setValidity` Flags

```typescript
// my-email-input.ts — extending the pattern with custom validation
class MyEmailInput extends MyTextInput {
  #validate(value: string): void {
    // First run parent validation (required, length, etc.)
    super['#validate']?.(value);

    // Add email-specific validation
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    const internals = (this as any)['#internals'] as ElementInternals;

    if (value && !emailRegex.test(value)) {
      internals.setValidity(
        { typeMismatch: true },
        'Please enter a valid email address.',
        (this as any)['#input']
      );
      const errorEl = this.shadowRoot!.querySelector('.error') as HTMLElement;
      errorEl.textContent = 'Please enter a valid email address.';
    }
  }
}

customElements.define('my-email-input', MyEmailInput, { extends: 'my-text-input' });
```

### 4.3 Complex Form Value: Object/JSON Submission

```typescript
// my-date-range.ts — Submitting structured data
class MyDateRange extends HTMLElement {
  static formAssociated = true;

  #internals: ElementInternals;
  #startInput: HTMLInputElement | null = null;
  #endInput: HTMLInputElement | null = null;

  constructor() {
    super();
    this.#internals = this.attachInternals();
    this.attachShadow({ mode: 'open' });
  }

  connectedCallback(): void {
    this.shadowRoot!.innerHTML = `
      <style>
        :host { display: flex; gap: 8px; align-items: center; }
        input { padding: 8px; border: 1px solid #ccc; border-radius: 4px; }
        span { color: #666; }
      </style>
      <input type="date" class="start" aria-label="Start date">
      <span>to</span>
      <input type="date" class="end" aria-label="End date">
    `;

    this.#startInput = this.shadowRoot!.querySelector('.start');
    this.#endInput = this.shadowRoot!.querySelector('.end');

    const sync = () => {
      const value = JSON.stringify({
        start: this.#startInput!.value,
        end: this.#endInput!.value,
      });
      // FormData will receive this as a single entry
      this.#internals.setFormValue(value);

      // Validate: end must be after start
      if (this.#startInput!.value && this.#endInput!.value &&
          this.#endInput!.value < this.#startInput!.value) {
        this.#internals.setValidity(
          { rangeUnderflow: true },
          'End date must be after start date.',
          this.#endInput!
        );
      } else {
        this.#internals.setValidity({});
      }
    };

    this.#startInput!.addEventListener('change', sync);
    this.#endInput!.addEventListener('change', sync);
  }

  get value(): { start: string; end: string } {
    return {
      start: this.#startInput?.value ?? '',
      end: this.#endInput?.value ?? '',
    };
  }
}

customElements.define('my-date-range', MyDateRange);
```

## 5. CSS Parts Theming

CSS Parts (`::part()`) allow consumers to style specific named elements inside a shadow tree from the outside. This is the standard mechanism for component theming without piercing shadow DOM boundaries.

### 5.1 Exposing Internal Elements as Parts

```typescript
// my-button.ts — exposing parts for theming
class MyButton extends HTMLElement {
  constructor() {
    super();
    this.attachShadow({ mode: 'open' });
  }

  connectedCallback(): void {
    const variant = this.getAttribute('variant') || 'primary';

    this.shadowRoot!.innerHTML = `
      <style>
        :host { display: inline-block; }
        .btn {
          display: inline-flex;
          align-items: center;
          gap: 8px;
          padding: 10px 20px;
          border: none;
          border-radius: 6px;
          font-size: 1rem;
          font-weight: 500;
          cursor: pointer;
          transition: background 0.2s, transform 0.1s;
          background: var(--btn-bg, #3b82f6);
          color: var(--btn-color, #fff);
        }
        .btn:hover { opacity: 0.9; }
        .btn:active { transform: scale(0.97); }
        .btn:disabled { opacity: 0.5; cursor: not-allowed; }
        .icon { display: inline-flex; align-items: center; }
      </style>
      <button class="btn" part="button" exportparts="icon">
        <span class="icon" part="icon">
          <slot name="icon"></slot>
        </span>
        <span part="label"><slot></slot></span>
      </button>
    `;
  }
}

customElements.define('my-button', MyButton);
```

## 6. SSR with Declarative Shadow DOM

Server-Side Rendering Web Components with Declarative Shadow DOM eliminates the "uncanny valley" where a component appears unstyled until JavaScript loads and hydrates it.

### 6.1 Server-Side Template Rendering (Node.js)

```typescript
// server/render.ts — SSR Web Component renderer
import { readFileSync } from 'node:fs';
import { join } from 'node:path';

interface ComponentConfig {
  tagName: string;
  templatePath: string;
  stylesPath?: string;
  props?: Record<string, string>;
  slots?: Record<string, string>;
  shadowMode?: 'open' | 'closed';
}

export function renderComponent(config: ComponentConfig): string {
  const { tagName, templatePath, stylesPath, props = {}, slots = {}, shadowMode = 'open' } = config;

  // Read template and styles
  const template = readFileSync(templatePath, 'utf-8');
  const styles = stylesPath ? readFileSync(stylesPath, 'utf-8') : '';

  // Build attributes string
  const attrs = Object.entries(props)
    .map(([key, value]) => `${key}="${escapeHtml(value)}"`)
    .join(' ');

  // Build slot content
  const slotContent = Object.entries(slots)
    .map(([name, content]) => {
      if (name === 'default') return content;
      return `<span slot="${name}">${content}</span>`;
    })
    .join('\n');

  return `
<${tagName} ${attrs}>
  <template shadowrootmode="${shadowMode}">
    ${styles ? `<style>${styles}</style>` : ''}
    ${template}
  </template>
  ${slotContent}
</${tagName}>`.trim();
}

function escapeHtml(str: string): string {
  return str
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
    .replace(/'/g, '&#039;');
}
```

### 6.2 Streaming SSR with DSD

```typescript
// server/stream-ssr.ts — Stream HTML with Declarative Shadow DOM
import { createReadStream } from 'node:fs';
import { Readable, Transform } from 'node:stream';

class DSDComponentStream extends Transform {
  private buffer = '';

  constructor(private components: Map<string, ComponentConfig>) {
    super();
  }

  _transform(chunk: Buffer, _encoding: string, callback: () => void): void {
    this.buffer += chunk.toString();

    // Replace component placeholders with DSD-rendered HTML
    // Pattern: <my-component data-props='{"key":"value"}'></my-component>
    const placeholderRegex = /<(\w[\w-]*)([^>]*)><\/\1>/g;

    this.buffer = this.buffer.replace(placeholderRegex, (match, tagName, attrsStr) => {
      const config = this.components.get(tagName);
      if (!config) return match; // Unknown component — leave as-is

      // Parse data-props JSON
      const dataPropsMatch = attrsStr.match(/data-props='([^']*)'/);
      const props: Record<string, string> = dataPropsMatch
        ? JSON.parse(dataPropsMatch[1])
        : {};

      return renderComponent({ ...config, props });
    });

    this.push(this.buffer);
    this.buffer = '';
    callback();
  }

  _flush(callback: () => void): void {
    if (this.buffer) {
      this.push(this.buffer);
    }
    callback();
  }
}

// Usage in an Express/Koa/Hono server:
import express from 'express';
const app = express();

const componentConfigs = new Map<string, ComponentConfig>();
componentConfigs.set('my-card', {
  tagName: 'my-card',
  templatePath: join(__dirname, '../components/my-card/template.html'),
  stylesPath: join(__dirname, '../components/my-card/styles.css'),
});

app.get('/', (_req, res) => {
  res.setHeader('Content-Type', 'text/html; charset=utf-8');

  const htmlStream = createReadStream(join(__dirname, '../templates/page.html'));
  const dsdStream = new DSDComponentStream(componentConfigs);

  htmlStream.pipe(dsdStream).pipe(res);
});
```

### 6.6 Hydration Mismatch Detection

```typescript
// client/hydrate.ts — Detect and report SSR/client mismatches
class HydrationValidator {
  static validate(root: Document | ShadowRoot = document): void {
    root.querySelectorAll('template[shadowrootmode]').forEach((template) => {
      const host = template.parentElement;
      if (!host) return;

      const tagName = host.tagName.toLowerCase();
      const customElement = customElements.get(tagName);

      if (!customElement) {
        console.warn(`[Hydration] Unknown custom element: <${tagName}>. Will upgrade when defined.`);
        return;
      }

      // Check if element is already upgraded
      if (host.constructor === customElement) {
        console.log(`[Hydration] <${tagName}> already upgraded with existing DSD shadow root.`);
      } else {
        console.log(`[Hydration] <${tagName}> pending upgrade. DSD shadow root will be reused.`);
      }

      // Validate slot structure
      const slots = template.content.querySelectorAll('slot');
      const usedSlotNames = new Set<string>();
      host.querySelectorAll(':scope > [slot]').forEach((slotted) => {
        usedSlotNames.add(slotted.getAttribute('slot') || '');
      });

      slots.forEach((slot) => {
        const name = slot.getAttribute('name') || 'default';
        if (!usedSlotNames.has(name) && name !== 'default') {
          console.warn(`[Hydration] <${tagName}>: slot "${name}" has no matching content.`);
        }
      });
    });
  }
}

// Run after DOM is ready
document.addEventListener('DOMContentLoaded', () => HydrationValidator.validate());

class MyFormCard extends HTMLElement {
  static formAssociated = true;

  #internals!: ElementInternals;
  #titleInput: HTMLInputElement | null = null;
  #descTextarea: HTMLTextAreaElement | null = null;

  // DSD template (used for SSR; also as fallback)
  static get dsdTemplate(): string {
    return `
      <style>
        :host {
          display: block;
          border: 1px solid var(--card-border, #e5e7eb);
          border-radius: var(--card-radius, 8px);
          background: var(--card-bg, #fff);
          padding: 20px;
          font-family: system-ui, sans-serif;
        }
        :host([invalid]) { border-color: #e11d48; }
        .form-group { margin-bottom: 12px; }
        label { display: block; font-size: 0.875rem; font-weight: 500; margin-bottom: 4px; color: #374151; }
        input, textarea {
          width: 100%;
          padding: 8px 12px;
          border: 1px solid #d1d5db;
          border-radius: 6px;
          font-size: 1rem;
          box-sizing: border-box;
          transition: border-color 0.15s;
        }
        input:focus, textarea:focus {
          outline: none;
          border-color: #3b82f6;
          box-shadow: 0 0 0 3px rgba(59,130,246,0.1);
        }
        .error { color: #e11d48; font-size: 0.75rem; margin-top: 4px; min-height: 1em; }
      </style>
      <div class="form-group">
        <label for="title">Title</label>
        <input type="text" id="title" part="title-input" placeholder="Enter title" required>
        <div class="error" part="error-message"></div>
      </div>
      <div class="form-group">
        <label for="desc">Description</label>
        <textarea id="desc" part="desc-textarea" rows="3" placeholder="Enter description"></textarea>
      </div>
    `;
  }

  constructor() {
    super();
    this.#internals = this.attachInternals();

    // Check for SSR-provided DSD shadow root
    if (!this.shadowRoot) {
      this.attachShadow({ mode: 'open' });
      this.shadowRoot!.innerHTML = MyFormCard.dsdTemplate;
    }
  }

  connectedCallback(): void {
    // If DSD was used, shadow root already has the template — just bind references
    this.#titleInput = this.shadowRoot!.querySelector('#title') as HTMLInputElement;
    this.#descTextarea = this.shadowRoot!.querySelector('#desc') as HTMLTextAreaElement;

    this.#titleInput!.addEventListener('input', () => this.#syncAndValidate());
    this.#descTextarea!.addEventListener('input', () => this.#syncAndValidate());

    // Initial sync
    this.#syncAndValidate();
  }

  #syncAndValidate(): void {
    const title = this.#titleInput!.value;
    const desc = this.#descTextarea!.value;
    const errorEl = this.shadowRoot!.querySelector('.error') as HTMLElement;

    // Set form value as structured JSON
    this.#internals.setFormValue(JSON.stringify({ title, description: desc }));

    // Validate
    if (!title.trim()) {
      this.#internals.setValidity(
        { valueMissing: true },
        'Title is required.',
        this.#titleInput!
      );
      errorEl.textContent = 'Title is required.';
    } else {
      this.#internals.setValidity({});
      errorEl.textContent = '';
    }
  }

  formResetCallback(): void {
    if (this.#titleInput) this.#titleInput.value = '';
    if (this.#descTextarea) this.#descTextarea.value = '';
    this.#syncAndValidate();
  }

  formStateRestoreCallback(state: string | null): void {
    if (state) {
      try {
        const { title, description } = JSON.parse(state);
        if (this.#titleInput) this.#titleInput.value = title || '';
        if (this.#descTextarea) this.#descTextarea.value = description || '';
        this.#syncAndValidate();
      } catch { /* ignore invalid state */ }
    }
  }
}

customElements.define('my-form-card', MyFormCard);
```

## Summary

This advanced Web Component blueprint covers five production-grade patterns:

1. **Declarative Shadow DOM** — Server-rendered shadow roots with zero FOUC, streaming support, and `adoptedStyleSheets` integration. The DSD polyfill covers Safari and Firefox.

2. **Scoped Element Registries** — Isolated custom element definitions per shadow tree, enabling safe micro-frontend composition. Polyfill essential for cross-browser support until the API lands broadly.

3. **Form-Associated Custom Elements with Validity** — Full participation in native HTML forms, Constraint Validation API integration via `ElementInternals`, `formResetCallback`, `formStateRestoreCallback`, and structured (JSON) form values.

4. **CSS Parts Theming** — `::part()` for consumer-controlled styling, `exportparts` for forwarding through component hierarchies, design token integration, and the future `::theme()` pseudo-element.

5. **SSR with Declarative Shadow DOM** — Server-side rendering via Node.js streams, Lit SSR integration, DSD polyfill for legacy browsers, and hydration mismatch detection. Edge-deployable via Workers.

All patterns are composable. The final example (`MyFormCard`) demonstrates a single component using all five techniques: DSD-rendered, form-associated with validation, CSS Parts-exposed, and SSR-ready with scoped registry support.
