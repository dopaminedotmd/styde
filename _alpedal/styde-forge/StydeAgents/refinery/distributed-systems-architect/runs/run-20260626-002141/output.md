DESIGN: Distributed Key-Value Store
CONSENSUS: Raft over gRPC. Three-node raft groups per shard. Log replication with majority quorum. Committed entries applied to state machine in order. Single-vote leader prevents split-brain. Leader leases for read-only queries avoid IO on followers.
LEADER ELECTION: Randomized election timeouts (150-300ms). PreVote phase prevents disruptive candidates on partitioned nodes. Leader steps down on election timeout or higher term discovery. Heartbeat interval 50ms. Stale leaders detected via monotonic term counter.
SHARDING: Consistent hashing on 256 virtual nodes per physical node. Key hash ring: SHA-256 mod 2^64. Virtual nodes distribute load evenly when nodes join/leave. Minimal reshuffling on topology change — only nearest neighbors rebalance. Lookup O(log N) via skip-list ring index.
REPLICATION: Synchronous replication within raft group (3 nodes). Async cross-region replication for disaster recovery. Read quorum: consistent reads hit leader only; stale reads hit any node (session-level consistency config). Write majority: 2 of 3 per shard. Anti-entropy gossip for eventual convergence across regions.
CAP: Partition tolerance enforced at raft layer. Consistency on writes via quorum (CP). When partition occurs, minority side halts writes — availability sacrificed for linearizability. Configurable per-key: CP mode (default) or AP mode (last-write-wins, async replication, risk of divergence).
FAILURE DETECTION: Accrual failure detector (phi-adaptive). Phi threshold = 8 triggers suspected state. Three consecutive rounds of lost heartbeats = confirmed dead. Dead node triggers raft leader election in affected shards. Re-replication to bring shard back to 3 nodes from spare pool.
CONSISTENCY MODEL: Linearizable by default (raft). Session consistency for read-after-write. Eventual consistency for cross-region async replicas. Causal consistency via vector clocks on AP keys.
