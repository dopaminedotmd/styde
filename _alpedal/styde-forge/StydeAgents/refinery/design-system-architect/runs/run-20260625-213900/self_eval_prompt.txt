## 1. Design Principles

| Principle | Description |
|-----------|-------------|
| **Clarity** | Interfaces must be immediately understandable. Use familiar patterns and clear visual hierarchy. |
| **Consistency** | Every component shares a single source of truth (design tokens). No one-off values. |
| **Efficiency** | Minimize cognitive load. Progressive disclosure. Shortest path to completion. |
| **Accessibility** | WCAG 2.2 AA compliance minimum. All components must be keyboard-navigable and screen-reader friendly. |
| **Resilience** | Components degrade gracefully. Handle empty, loading, error, and extreme-content states. |
| **Theme-Agnostic** | Every component must render correctly in both light and dark themes without layout shifts. |

## 3. Theming Strategy

### 3.1 Light Theme

The default theme. Activated when `data-theme="light"` or no `data-theme` set on `<html>`.

**Characteristics:**
- High contrast ratios (minimum 4.5:1 body, 3:1 large text)
- Bright backgrounds with dark text
- Subtle gray borders and shadows
- Blue-600 brand accents

**Key Background Stack:**

```
Page bg:        gray-50  (#f9fafb)
Card bg:        white    (#ffffff)
Input bg:       white    (#ffffff)
Elevated bg:    white    (#ffffff)
```

### 3.2 Dark Theme

Activated when `data-theme="dark"` on `<html>`.

**Characteristics:**
- Dark backgrounds (`#0f1117` page, `#111827` card)
- Light text (`#f9fafb` primary)
- Softer borders (`gray-700`)
- Deeper shadows with higher opacity
- Brand colors maintain saturation; blue-400 for links

**Key Background Stack:**

```
Page bg:        gray-950 (#030712)
Card bg:        gray-900 (#111827)
Input bg:       gray-800 (#1f2937)
Elevated bg:    gray-800 (#1f2937)
```

**Color Adjustments in Dark Mode:**
- Gray palette is inverted (50 Ōåö 950, 100 Ōåö 900, etc.)
- Blue, red, green, yellow mid-tones (500) remain unchanged for brand recognition
- Semantic bg tokens switch to darker variants
- Text tokens invert: `text-primary` maps to `gray-50` instead of `gray-900`

### 3.3 Theme Switching Mechanism

```
Implementation Strategy: CSS Custom Properties + data-theme attribute
```

**Architecture:**

1. **Token Layer (CSS):** All components reference `--sty-*` tokens exclusively ŌĆö never hardcoded hex values.
2. **Theme Stylesheet:** Two blocks: `:root, [data-theme="light"]` and `[data-theme="dark"]` defining the same tokens with different primitive references.
3. **Runtime Switching:**
   ```js
   // Set theme
   document.documentElement.setAttribute('data-theme', 'dark');
   // Persist
   localStorage.setItem('sty-theme', 'dark');
   // Respect OS preference on first visit
   const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
   const stored = localStorage.getItem('sty-theme');
   const theme = stored ?? (prefersDark ? 'dark' : 'light');
   ```
4. **Transition:** Apply a brief `transition: background-color 200ms ease, color 200ms ease` on `:root` to avoid jarring flashes.
5. **System Sync:** Listen for `matchMedia` changes to update theme when OS preference changes (if user hasn't manually set a preference).

## 5. Component Usage Guidelines

### 5.1 Composition Patterns

**Rule: Prefer composition over configuration.** Components expose slot props (`header`, `footer`, `leftElement`, `rightElement`) rather than bloated prop surfaces.

**Ō£ģ DO:**
```jsx
<Card header={<CardTitle>Revenue</CardTitle>}>
  <Chart data={revenueData} />
</Card>
```

**ŌØī DON'T:**
```jsx
<Card title="Revenue" chartData={revenueData} />
```

**Pattern: Controlled & Uncontrolled.** Every stateful component supports both modes:

```jsx
// Controlled
<Modal isOpen={isOpen} onClose={() => setIsOpen(false)} />

// Uncontrolled (via defaultIsOpen)
<Dropdown defaultIsOpen={false} trigger={<Button>Menu</Button>}>
  <DropdownItem>Action</DropdownItem>
</Dropdown>
```

**Pattern: Polymorphic components.** Where appropriate, components support an `as` prop:

```jsx
<Button as="a" href="/dashboard">Go to Dashboard</Button>
<Card as="section" aria-label="User profile">...</Card>
```

### 5.2 Accessibility Requirements

All components must meet these baseline requirements:

| Requirement | Implementation |
|-------------|----------------|
| **Keyboard navigation** | Tab/Shift+Tab for focus order. Arrow keys for composite widgets (tabs, dropdown, table sort). Enter/Space to activate. Escape to dismiss overlays. |
| **Focus management** | Modals/Dropdowns trap focus. Focus returns to trigger on close. Focus rings visible (`outline: 2px solid --sty-color-border-focus` with `2px` offset). |
| **ARIA attributes** | All interactive elements have appropriate roles, labels, and states. Generated IDs for `aria-labelledby` / `aria-describedby` relationships. |
| **Screen reader announcements** | Dynamic content changes (toast, modal open, loading complete) use `aria-live` regions (`polite` for info, `assertive` for errors). |
| **Color contrast** | Minimum 4.5:1 for normal text, 3:1 for large text (18px+ bold or 24px+ regular), 3:1 for UI components and graphical objects. |
| **Reduced motion** | Respect `prefers-reduced-motion: reduce`. Disable animations, use instant transitions. |
| **Touch targets** | Minimum 44├Ś44px interactive area (WCAG 2.5.5). All buttons, dropdown items, tab triggers meet this. |

**Focus Ring:**

```
:focus-visible {
  outline: 2px solid var(--sty-color-border-focus);
  outline-offset: 2px;
  border-radius: var(--sty-radius-md);
}
```

### 5.3 Responsive Behavior

**Mobile-first approach:** Base styles target small viewports. Breakpoints add overrides for larger screens.

| Component | Mobile Behavior |
|-----------|----------------|
| **Modal** | Fullscreen at `sm` breakpoint, centered dialog above |
| **Table** | Horizontal scroll wrapper at `sm`; optional card layout (each row becomes a card) at `xs` |
| **Tabs** | Horizontal scroll if tabs overflow; `overflow="scroll"` |
| **Dropdown** | On narrow viewports, may render as a bottom sheet |
| **Toast** | Full-width at `sm`, positioned blocks above |
| **Card** | Stacks vertically in grids; `--sty-container-*` constrains max-width |

**Breakpoint Usage in Components:**

```
/* Base: mobile */
.card-grid { display: grid; grid-template-columns: 1fr; gap: var(--sty-space-4); }

/* Tablet + */
@media (min-width: 768px) {
  .card-grid { grid-template-columns: repeat(2, 1fr); }
}

/* Desktop + */
@media (min-width: 1024px) {
  .card-grid { grid-template-columns: repeat(3, 1fr); }
}
```

### 5.4 State Management

Every interactive component must handle these UI states explicitly:

| State | Definition | Example |
|-------|-----------|---------|
| **Default** | Normal resting state | Button with label |
| **Hover** | Pointer over element | Button background shifts |
| **Focus** | Keyboard focused | Focus ring appears |
| **Active/Pressed** | Being clicked/pressed | Slightly darker background |
| **Disabled** | Non-interactive | Reduced opacity, `cursor: not-allowed` |
| **Loading** | Operation in progress | Spinner, skeleton, or progress bar |
| **Empty** | No data to display | Empty state illustration + message |
| **Error** | Operation failed | Error message + retry action |
| **Selected/Active** | Currently chosen item | Highlighted row, active tab indicator |
| **Expanded** | Expanded accordion/dropdown | Rotated chevron |

**Loading States Priority:**

1. **Skeleton screens** for initial page/component load
2. **Inline spinners** for button actions (submit, save)
3. **Progress bars** for multi-step processes
4. **Optimistic UI** where safe (toggle switches, checkbox states)

- `--sty-color-bg-brand-hover`
- `--sty-text-lg`
- `--sty-shadow-md`

### 6.3 Tailwind CSS Integration

For projects using Tailwind, tokens map to the theme extension:

```js
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        // Primitive
        gray: { 50: 'var(--sty-color-gray-50)', /* ... */ },
        blue: { 50: 'var(--sty-color-blue-50)', /* ... */ },
        // Semantic
        brand: {
          DEFAULT: 'var(--sty-color-bg-brand)',
          hover: 'var(--sty-color-bg-brand-hover)',
        },
      },
      spacing: {
        'sty-1': 'var(--sty-space-1)',
        // ...
      },
      fontSize: {
        'sty-xs': ['var(--sty-text-xs)', { lineHeight: '1rem' }],
        // ...
      },
      boxShadow: {
        'sty-sm': 'var(--sty-shadow-sm)',
        // ...
      },
      borderRadius: {
        'sty-md': 'var(--sty-radius-md)',
        // ...
      },
    },
  },
};
```
