## 1. Architecture Overview

### Testing Pyramid

```
         ╱ E2E ╲          Playwright — slow, expensive, real browser
        ╱───────╲
       ╱Integration╲       Vitest + Testing Library — medium speed, component trees
      ╱─────────────╲
     ╱   Unit Tests   ╲    Vitest — fast, isolated functions/hooks
    ╱───────────────────╲
```

### Technology Choices & Rationale

| Layer | Tool | Rationale |
|---|---|---|
| **Test Runner** | Vitest | Native Vite integration, Jest-compatible API, HMR in watch mode, TypeScript out of the box |
| **Component Testing** | Testing Library (React) | User-centric queries, accessibility-first, discourages implementation-detail testing |
| **E2E Testing** | Playwright | Multi-browser, auto-wait, trace viewer, parallel sharding, first-class TypeScript |
| **API Mocking** | MSW (Mock Service Worker) | Intercepts at network layer, works identically in Node and browser, reusable handlers |
| **Visual Regression** | Playwright screenshot assertions | Built-in `toHaveScreenshot()` with pixelmatch diffing |
| **Coverage** | v8 provider (Vitest) | Native V8 coverage, faster than Istanbul on large codebases |
| **CI** | GitHub Actions | Ubiquitous, matrix builds, artifact storage, integration with Playwright dashboard |

```typescript
// src/hooks/useDebounce.test.ts
// ============================================================
import { renderHook, act } from '@testing-library/react';
import { describe, it, expect, vi, beforeEach, afterEach } from 'vitest';
import { useDebounce } from './useDebounce';

describe('useDebounce', () => {
  beforeEach(() => {
    vi.useFakeTimers();
  });

  afterEach(() => {
    vi.useRealTimers();
  });

  it('returns initial value immediately', () => {
    const { result } = renderHook(
      ({ value }) => useDebounce(value, 300),
      { initialProps: { value: 'hello' } }
    );

    expect(result.current).toBe('hello');
  });

  it('debounces value changes', () => {
    const { result, rerender } = renderHook(
      ({ value }) => useDebounce(value, 300),
      { initialProps: { value: 'hello' } }
    );

    // Change value — debounced value should NOT update yet
    rerender({ value: 'hello world' });
    expect(result.current).toBe('hello'); // Still old value

    // Fast-forward past debounce delay
    act(() => {
      vi.advanceTimersByTime(300);
    });
    expect(result.current).toBe('hello world');
  });

  it('cancels previous timer on rapid updates', () => {
    const { result, rerender } = renderHook(
      ({ value }) => useDebounce(value, 500),
      { initialProps: { value: 'a' } }
    );

    // Type characters rapidly
    rerender({ value: 'ab' });
    act(() => { vi.advanceTimersByTime(200); });

    rerender({ value: 'abc' });
    act(() => { vi.advanceTimersByTime(200); });

    rerender({ value: 'abcd' });

    // Before delay: still first value
    expect(result.current).toBe('a');

    // After full delay: only the last value
    act(() => { vi.advanceTimersByTime(500); });
    expect(result.current).toBe('abcd');
  });

  it('cleans up timeout on unmount', () => {
    const clearTimeoutSpy = vi.spyOn(global, 'clearTimeout');
    const { unmount } = renderHook(() => useDebounce('test', 300));

    unmount();
    expect(clearTimeoutSpy).toHaveBeenCalled();
    clearTimeoutSpy.mockRestore();
  });
});
```

## 3. Testing Library Patterns for React Components

### 3.1 Testing Principles

```
┌─────────────────────────────────────────────────────────────┐
│  TEST BEHAVIOR, NOT IMPLEMENTATION                          │
│                                                             │
│  ✓ "When the user clicks Save, a success toast appears"     │
│  ✗ "The onClick handler dispatches SAVE_SUCCESS"            │
│                                                             │
│  Query priority:                                            │
│  1. getByRole (most accessible, most resilient)             │
│  2. getByLabelText (form fields)                            │
│  3. getByPlaceholderText                                    │
│  4. getByText (visible text)                                │
│  5. getByDisplayValue (filled form fields)                  │
│  6. getByAltText (images)                                   │
│  7. getByTitle                                              │
│  8. getByTestId (last resort — fragile)                     │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 Test Utilities (`tests/utils/test-utils.tsx`)

```typescript
// ============================================================
// tests/utils/test-utils.tsx — Custom render with providers
// ============================================================
import React, { type ReactElement } from 'react';
import { render, type RenderOptions } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
import { BrowserRouter } from 'react-router-dom';
import { ThemeProvider } from '@/components/ThemeProvider';
import { Toaster } from '@/components/Toaster';

// ── Default wrapper with all providers ──────────────────────
interface AllTheProvidersProps {
  children: React.ReactNode;
  route?: string;
  queryClient?: QueryClient;
}

function AllTheProviders({
  children,
  route = '/',
  queryClient: customQueryClient,
}: AllTheProvidersProps) {
  const queryClient =
    customQueryClient ??
    new QueryClient({
      defaultOptions: {
        queries: {
          retry: false,        // No retries in tests
          gcTime: 0,           // No garbage collection delay
        },
        mutations: {
          retry: false,
        },
      },
    });

  // Set initial route for Router
  window.history.pushState({}, 'Test page', route);

  return (
    <QueryClientProvider client={queryClient}>
      <ThemeProvider defaultTheme="light">
        <BrowserRouter>
          {children}
        </BrowserRouter>
        <Toaster />
      </ThemeProvider>
    </QueryClientProvider>
  );
}

// ── Custom render ───────────────────────────────────────────
interface CustomRenderOptions extends Omit<RenderOptions, 'wrapper'> {
  route?: string;
  queryClient?: QueryClient;
  user?: ReturnType<typeof userEvent.setup>;
}

export function renderWithProviders(
  ui: ReactElement,
  options: CustomRenderOptions = {}
) {
  const { route, queryClient, ...renderOptions } = options;

  const Wrapper = ({ children }: { children: React.ReactNode }) => (
    <AllTheProviders route={route} queryClient={queryClient}>
      {children}
    </AllTheProviders>
  );

  // Always use userEvent.setup() — never fireEvent directly
  const user = userEvent.setup();

  return {
    user,
    ...render(ui, { wrapper: Wrapper, ...renderOptions }),
  };
}

// ── Re-export everything ────────────────────────────────────
export * from '@testing-library/react';
export { renderWithProviders as render };
```

```typescript
// ============================================================
// Patterns for async testing
// ============================================================

// ── Pattern 1: findBy* queries (retry until found) ──────────
it('loads and displays user data', async () => {
  render(<UserProfile userId="1" />);

  // findBy* queries wait for the element to appear (default: 1000ms)
  const name = await screen.findByText('John Doe');
  expect(name).toBeInTheDocument();

  // findByRole also works
  expect(await screen.findByRole('heading', { name: /john doe/i })).toBeInTheDocument();
});

// ── Pattern 2: waitFor for complex assertions ───────────────
it('updates list after mutation', async () => {
  const { user } = render(<TodoList />);

  await user.type(screen.getByRole('textbox', { name: /new todo/i }), 'Buy milk');
  await user.click(screen.getByRole('button', { name: /add/i }));

  await waitFor(() => {
    const listItems = screen.getAllByRole('listitem');
    expect(listItems).toHaveLength(1);
    expect(listItems[0]).toHaveTextContent('Buy milk');
  });
});

// ── Pattern 3: waitForElementToBeRemoved ────────────────────
it('removes loading spinner after data loads', async () => {
  render(<DataTable />);

  // Assert spinner exists initially
  expect(screen.getByRole('progressbar')).toBeInTheDocument();

  // Wait for it to disappear
  await waitForElementToBeRemoved(() => screen.queryByRole('progressbar'));

  // Now assert data is visible
  expect(screen.getByRole('table')).toBeInTheDocument();
});

// ── Pattern 4: within() for scoped queries ──────────────────
it('scopes queries to a specific section', async () => {
  render(<Dashboard />);

  const sidebar = screen.getByRole('complementary', { name: /sidebar/i });
```

```typescript
import { type FullConfig } from '@playwright/test';
import fs from 'node:fs';
import path from 'node:path';

async function globalSetup(config: FullConfig) {
  // ── Ensure test directories exist ─────────────────────────
  const dirs = ['test-results', 'playwright-report'];
  for (const dir of dirs) {
    const dirPath = path.join(config.rootDir, '..', dir);
    if (!fs.existsSync(dirPath)) {
      fs.mkdirSync(dirPath, { recursive: true });
    }
  }

  // ── Seed test database or set up fixtures ─────────────────
  // Example: call a setup API on the backend
  const baseURL = config.projects[0]?.use.baseURL;
  if (baseURL) {
    try {
      const response = await fetch(`${baseURL}/api/test/setup`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ seed: true }),
      });
      console.log('Test data seeded:', response.status);
    } catch {
      console.warn('⚠ Could not connect to backend for test setup');
    }
  }
}
```

### 4.3 Base Page Object (`e2e/pages/BasePage.ts`)

```typescript
// ============================================================
// e2e/pages/BasePage.ts — Base Page Object Model
// ============================================================
import { type Page, type Locator, expect } from '@playwright/test';

/**
 * Base Page Object — all page objects extend this.
 *
 * Provides shared utilities: navigation, waiting, common assertions,
 * toast/snackbar handling, and cookie/localStorage helpers.
 */
export abstract class BasePage {
  protected readonly page: Page;

  constructor(page: Page) {
    this.page = page;
  }

  // ── URL Helpers ───────────────────────────────────────────

  /** Navigate to a path relative to baseURL */
  async goto(path: string = '/'): Promise<void> {
    await this.page.goto(path);
  }

  /** Assert current URL matches exactly */
  async shouldBeOnPath(path: string): Promise<void> {
    await expect(this.page).toHaveURL(new RegExp(`${path}$`));
  }

  /** Wait for the page to be fully loaded (network idle) */
  async waitForPageLoad(): Promise<void> {
    await this.page.waitForLoadState('networkidle');
  }

  // ── Common Locators ───────────────────────────────────────

  /** Main content area */
  get mainContent(): Locator {
    return this.page.getByRole('main');
  }

  /** Page heading (h1) */
  get heading(): Locator {
    return this.page.getByRole('heading', { level: 1 });
  }

  /** Toast/snackbar notification */
  get toast(): Locator {
    return this.page.getByRole('alert').or(
      this.page.locator('[data-testid="toast"]')
    );
  }

  /** Loading spinner */
  get loadingSpinner(): Locator {
    return this.page.getByRole('progressbar').or(
      this.page.locator('[data-testid="loading-spinner"]')
    );
  }

  /** Error message banner */
  get errorBanner(): Locator {
    return this.page.getByRole('alert').filter({ hasText: /error/i });
  }

  /** Global navigation */
  get navigation(): Locator {
    return this.page.getByRole('navigation');
  }

  // ── Action Helpers ────────────────────────────────────────

  /** Wait for loading to finish */
  async waitForLoadComplete(): Promise<void> {
    // Wait for spinners to disappear or network to settle
    await Promise.race([
      this.loadingSpinner.waitFor({ state: 'hidden', timeout: 10_000 })
        .catch(() => {}),
      this.page.waitForLoadState('networkidle', { timeout: 10_000 })
        .catch(() => {}),
    ]);
  }

  /** Assert a success toast appears with given text */
  async expectSuccessToast(message: string): Promise<void> {
    await expect(this.toast).toBeVisible();
    await expect(this.toast).toContainText(message);
  }

  /** Dismiss a toast if dismissible */
  async dismissToast(): Promise<void> {
    const dismissButton = this.toast.getByRole('button', { name: /close|dismiss/i });
    if (await dismissButton.isVisible()) {
      await dismissButton.click();
    }
  }

  /** Take a named screenshot for debugging */
  async screenshot(name: string): Promise<void> {
    await this.page.screenshot({ path: `test-results/screenshots/${name}.png` });
  }

  // ── Accessibility ─────────────────────────────────────────

  /** Run axe-core accessibility scan (requires @axe-core/playwright) */
  async a11yScan(): Promise<void> {
    // Dynamically import to avoid requiring the package for all tests
    try {
      const { default: AxeBuilder } = await import('@axe-core/playwright');
      const results = await new AxeBuilder({ page: this.page }).analyze();
      expect(results.violations).toEqual([]);
    } catch {
      // axe-core/playwright not installed — skip
      console.log('Skipping a11y scan (@axe-core/playwright not installed)');
    }
  }

  // ── Local Storage / Cookies ───────────────────────────────

  async setLocalStorage(key: string, value: string): Promise<void> {
    await this.page.evaluate(
      ({ k, v }) => localStorage.setItem(k, v),
      { k: key, v: value }
    );
  }

  async getLocalStorage(key: string): Promise<string | null> {
    return this.page.evaluate(
      ({ k }) => localStorage.getItem(k),
      { k: key }
    );
  }

  // ── Dialogs ───────────────────────────────────────────────

  /** Accept the next browser dialog (alert, confirm, prompt) */
  async acceptDialog(): Promise<void> {
    this.page.once('dialog', async (dialog) => {
      await dialog.accept();
    });
  }
}
```

```typescript
// ============================================================
// tests/mocks/handlers/auth.ts — Auth API handlers
// ============================================================
import { http, HttpResponse, delay } from 'msw';

/**
 * Auth API handlers — intercept all /api/auth/* calls.
 *
 * Each handler returns realistic responses with optional delay
 * to simulate network latency in tests.
 */

export const authHandlers = [
  // ── POST /api/auth/login ─────────────────────────────────
  http.post('*/api/auth/login', async ({ request }) => {
    const body = (await request.json()) as { email: string; password: string };

    // Simulate network latency (remove in production tests)
    await delay(100);

    if (body.email === 'user@example.com' && body.password === 'correct-password') {
      return HttpResponse.json(
        {
          token: 'mock-jwt-token-for-testing',
          user: {
            id: 'usr_test123',
            email: 'user@example.com',
            name: 'Test User',
            role: 'user',
          },
          expiresAt: new Date(Date.now() + 3600_000).toISOString(),
        },
        { status: 200 }
      );
    }

    // Invalid credentials
    return HttpResponse.json(
      { error: 'Invalid email or password', code: 'AUTH_INVALID_CREDENTIALS' },
      { status: 401 }
    );
  }),

  // ── POST /api/auth/register ──────────────────────────────
  http.post('*/api/auth/register', async ({ request }) => {
    const body = (await request.json()) as {
      email: string;
      password: string;
      name: string;
    };

    await delay(150);

    // Simulate duplicate email
    if (body.email === 'existing@example.com') {
      return HttpResponse.json(
        { error: 'Email already registered', code: 'AUTH_DUPLICATE_EMAIL' },
        { status: 409 }
      );
    }

    return HttpResponse.json(
      {
        token: 'mock-jwt-token-new-user',
        user: {
          id: `usr_${Date.now()}`,
          email: body.email,
          name: body.name,
          role: 'user',
        },
      },
      { status: 201 }
    );
  }),

  // ── GET /api/auth/me ─────────────────────────────────────
  http.get('*/api/auth/me', async ({ request }) => {
    const authHeader = request.headers.get('Authorization');

    if (!authHeader || !authHeader.startsWith('Bearer ')) {
      return HttpResponse.json(
        { error: 'Unauthorized', code: 'AUTH_MISSING_TOKEN' },
        { status: 401 }
      );
    }

    const token = authHeader.replace('Bearer ', '');

    if (token === 'mock-jwt-token-for-testing') {
      return HttpResponse.json({
        id: 'usr_test123',
        email: 'user@example.com',
        name: 'Test User',
        role: 'user',
      });
    }

    // Expired/invalid token
    return HttpResponse.json(
      { error: 'Token expired or invalid', code: 'AUTH_INVALID_TOKEN' },
      { status: 401 }
    );
  }),

  // ── POST /api/auth/logout ────────────────────────────────
  http.post('*/api/auth/logout', () => {
    return HttpResponse.json({ success: true }, { status: 200 });
  }),

  // ── POST /api/auth/refresh ───────────────────────────────
  http.post('*/api/auth/refresh', () => {
    return HttpResponse.json({
      token: 'mock-jwt-token-refreshed',
      expiresAt: new Date(Date.now() + 3600_000).toISOString(),
    });
  }),
];
```

### 5.3 Handler Definitions (`tests/mocks/handlers/users.ts`)

```typescript
// ============================================================
// tests/mocks/handlers/users.ts — Users API handlers
// ============================================================
import { http, HttpResponse, delay } from 'msw';

const mockUsers = [
  { id: '1', name: 'Alice Johnson', email: 'alice@example.com', role: 'admin' },
  { id: '2', name: 'Bob Smith', email: 'bob@example.com', role: 'editor' },
  { id: '3', name: 'Carol White', email: 'carol@example.com', role: 'viewer' },
];

export const userHandlers = [
  // ── GET /api/users — List users ───────────────────────────
  http.get('*/api/users', async ({ request }) => {
    await delay(200);

    const url = new URL(request.url);
    const search = url.searchParams.get('q')?.toLowerCase();
    const page = Number(url.searchParams.get('page')) || 1;
    const limit = Number(url.searchParams.get('limit')) || 10;

    let filtered = mockUsers;
    if (search) {
      filtered = mockUsers.filter(
        (u) =>
          u.name.toLowerCase().includes(search) ||
          u.email.toLowerCase().includes(search)
      );
    }

    const start = (page - 1) * limit;
    const paginated = filtered.slice(start, start + limit);

    return HttpResponse.json({
      data: paginated,
      pagination: {
        page,
        limit,
        total: filtered.length,
        totalPages: Math.ceil(filtered.length / limit),
      },
    });
  }),

  // ── GET /api/users/:id — Single user ─────────────────────
  http.get('*/api/users/:id', async ({ params }) => {
    await delay(100);
    const user = mockUsers.find((u) => u.id === params.id);

    if (!user) {
      return HttpResponse.json(
        { error: 'User not found', code: 'USER_NOT_FOUND' },
        { status: 404 }
      );
    }

    return HttpResponse.json({ data: user });
  }),

  // ── POST /api/users — Create user ────────────────────────
  http.post('*/api/users', async ({ request }) => {
    await delay(150);
    const body = (await request.json()) as {
      name: string;
      email: string;
      role: string;
    };

    const newUser = {
      id: `usr_${Date.now()}`,
      ...body,
    };

    mockUsers.push(newUser);

    return HttpResponse.json({ data: newUser }, { status: 201 });
  }),

  // ── PATCH /api/users/:id — Update user ───────────────────
  http.patch('*/api/users/:id', async ({ params, request }) => {
    await delay(100);
    const body = (await request.json()) as Record<string, unknown>;
    const user = mockUsers.find((u) => u.id === params.id);

    if (!user) {
      return HttpResponse.json(
        { error: 'User not found' },
        { status: 404 }
      );
    }

    Object.assign(user, body);
    return HttpResponse.json({ data: user });
  }),

  // ── DELETE /api/users/:id — Delete user ──────────────────
  http.delete('*/api/users/:id', async ({ params }) => {
    await delay(100);
    const index = mockUsers.findIndex((u) => u.id === params.id);

    if (index === -1) {
      return HttpResponse.json(
        { error: 'User not found' },
        { status: 404 }
      );
    }

    mockUsers.splice(index, 1);
    return HttpResponse.json({ success: true }, { status: 200 });
  }),
];
```

## 7. Troubleshooting & Common Issues

### 7.1 Vitest

| Problem | Cause | Solution |
|---|---|---|
| `ReferenceError: document is not defined` | Test environment is `node`, not `jsdom` | Set `environment: 'jsdom'` in vitest config |
| Tests pass locally but fail in CI | Timezone, locale, or env differences | Set `process.env.TZ = 'UTC'` in global setup |
| `vi.mock()` not working | Mock must be at top level (hoisted) | Move `vi.mock()` before any imports, or use `vi.hoisted()` |
| Slow tests in watch mode | Too many expensive setup operations | Profile with `vitest --pool forks --poolOptions.forks.singleFork` |
| Coverage thresholds too strict for new projects | Unrealistic 80% on greenfield | Use `watermarks` for visual feedback without hard failure |

### 7.2 Testing Library

| Problem | Cause | Solution |
|---|---|---|
| `Unable to find role="button"` | Element is not in DOM or async renders haven't resolved | Use `findByRole` (async) instead of `getByRole` |
| `act(...)` warning | State update after unmount | Ensure `cleanup()` runs between tests |
| `Warning: An update to X inside a test was not wrapped in act(...)` | Async state update outside act | Use `waitFor` or `findBy*` queries |
| Can't find element by text | Text is split across multiple elements | Use `{ exact: false }` option or regex |

### 7.3 Playwright

| Problem | Cause | Solution |
|---|---|---|
| `waitForLoadState('networkidle')` timeout | Persistent WebSocket or polling connections | Use `waitForURL` or `waitForSelector` instead |
| Screenshot mismatch in CI | Different OS rendering (fonts, anti-aliasing) | Use `maxDiffPixelRatio: 0.02`, run on same OS in CI |
| Tests flaky on CI | Race conditions, timing issues | Add explicit `waitFor*`, use `retries: 2` |
| `browserType.launch: Executable doesn't exist` | Playwright browsers not installed | Run `npx playwright install --with-deps` |

### 7.4 MSW

| Problem | Cause | Solution |
|---|---|---|
| `[MSW] Warning: intercepted a request without a matching handler` | No handler defined for that endpoint | Add handler or use `onUnhandledRequest: 'bypass'` |
| Handlers not being called | Server not started, or URL pattern mismatch | Use `*` wildcard in handler URL: `*/api/users` |
| `server.use()` not taking effect | Called after request already fired | Call `server.use()` before triggering the request |
| Test data leaking between tests | Shared mutable state in handlers | Use factory functions, reset state in `afterEach` |

## Appendix A: Quick-Start Checklist

- [ ] Install dependencies: `pnpm add -D vitest @vitest/coverage-v8 @vitest/ui jsdom`
- [ ] Install Testing Library: `pnpm add -D @testing-library/react @testing-library/jest-dom @testing-library/user-event`
- [ ] Install Playwright: `pnpm add -D @playwright/test && npx playwright install --with-deps`
- [ ] Install MSW: `pnpm add -D msw`
- [ ] Create `vitest.config.ts` with jsdom environment
- [ ] Create `tests/setup.ts` and `tests/setup-dom.ts`
- [ ] Create `tests/utils/test-utils.tsx` with custom render
- [ ] Create MSW handlers under `tests/mocks/handlers/`
- [ ] Create MSW `server.ts` and add to `setupFiles`
- [ ] Create `e2e/playwright.config.ts`
- [ ] Create page objects under `e2e/pages/`
- [ ] Write first E2E spec under `e2e/specs/`
- [ ] Create `.github/workflows/test.yml` pipeline
- [ ] Add test scripts to `package.json`
- [ ] Run `pnpm test:run` to verify everything works

## Appendix B: Key Design Decisions

1. **jsdom over happy-dom**: jsdom has better compatibility with Testing Library and broader web API support, despite being slower. Use `happy-dom` only for pure unit tests that don't need full DOM.

2. **MSW over jest.mock/fetch-mock**: MSW operates at the network request level, meaning the same handlers work in tests, development, and Storybook. No mocking implementation details.

3. **Page Objects over inline locators**: While Playwright's locators are already quite readable, Page Objects provide a single place to update selectors when the UI changes, and they make specs read like a user manual.

4. **Sharded CI**: Splitting E2E tests across 3 shards per browser keeps individual job times under 10 minutes even with large test suites.

5. **Coverage on PRs only**: Generating and posting coverage reports only on pull requests avoids slowing down trunk pushes while maintaining quality gates where they matter most.

6. **Separate integration config**: Integration tests have different characteristics (slower, sequential, need MSW) and different coverage expectations than unit tests. A separate config avoids polluting the unit test setup.
