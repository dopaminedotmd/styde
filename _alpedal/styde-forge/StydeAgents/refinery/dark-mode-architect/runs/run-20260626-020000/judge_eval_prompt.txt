## Runtime Theme Switching

### API (`ThemeEngine` global)

```javascript
// Set theme (persists to localStorage)
ThemeEngine.setTheme('dark');

// Get current theme
ThemeEngine.getTheme(); // → 'dark'

// Cycle through available themes
ThemeEngine.cycleTheme(); // light → dark → oled → forest → light

// List available themes
ThemeEngine.getAvailableThemes(); // → ['light', 'dark', 'oled', 'forest']

// Listen to theme changes
const unsub = ThemeEngine.on('theme-change', ({ theme, previousTheme, source }) => {
  console.log(`Switched to ${theme} via ${source}`);
});

// Data viz helpers
ThemeEngine.getDataVizColors(6);  // → ['#74c0fc', '#69db7c', ...]
ThemeEngine.getDataVizCSSVar(3);  // → 'var(--dv-cat-03)'
ThemeEngine.resolveCSSVar('--surface-card'); // → '#25262b' (in dark mode)
```

### Events

| Event | Payload | When |
|-------|---------|------|
| `theme-init` | `{ theme, source }` | Engine bootstraps |
| `theme-change` | `{ theme, previousTheme, source }` | Theme changes |
| `print-start` | — | `window.matchMedia('print')` activates |
| `print-end` | — | Print dialog closes |

### Persistence Strategy

1. **localStorage key:** `styde-theme-preference`
2. **Format:** `{ theme: 'dark', version: 2, updatedAt: 1719360000000 }`
3. **Backward compatible:** Legacy flat string values auto-migrate
4. **Fallback chain:** stored preference → `prefers-color-scheme` → `light` default
5. **System preference live-tracking:** If user hasn't set an explicit preference, the engine follows OS-level dark/light changes

### Anti-FOUC (Flash of Unstyled Content)

The engine includes a synchronous inline script (placed in `<head>` before CSS) that reads `localStorage` and sets `data-theme` on `<html>` before the first paint. No flash occurs.

## Theme-Aware Data Visualization

Each theme defines 12 categorical colors (`--dv-cat-01` through `--dv-cat-12`) plus grid/axis/background:

## Print Theme

The print theme activates automatically via `@media print` and overrides **all** custom properties regardless of the active data-theme:

- **Backgrounds:** forced to white
- **Text:** forced to black/grays
- **Shadows:** removed entirely (`none`)
- **Border radius:** zeroed (sharp edges print cleaner)
- **Data viz colors:** grayscale ramp
- **Links:** standard browser underline colors

No JavaScript dependency — pure CSS media query.

## Cross-Cutting Concerns

- **`prefers-reduced-motion`**: Transition class (`data-theme-transitioning`) applies smooth 300ms crossfade only when the user hasn't requested reduced motion. When reduced motion is preferred, transition is set to `none`.
- **`color-scheme`**: Each theme sets the CSS `color-scheme` property (`light` or `dark`) so native browser UI (scrollbars, form controls) matches.
- **Focus rings**: Use `color-mix()` to derive a 25% opacity focus ring from the theme's `--border-focus` color, avoiding hardcoded values.
- **No runtime dependencies**: The CSS file works standalone. The JS engine is ~10KB unminified with zero external dependencies.
- **Graceful degradation**: Without JS, the page renders in `light` theme. With JS disabled, the inline anti-FOUC script still applies the stored theme since it runs before the JS engine.
