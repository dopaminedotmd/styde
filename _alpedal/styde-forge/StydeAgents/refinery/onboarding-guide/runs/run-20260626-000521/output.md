ONBOARDING PLAN: SOFTWARE ENGINEER — MID-LEVEL / TECHNOLOGY DEPT
WEEK 1 — FOUNDATIONS
Day 1-2: Tools & Access
Set up laptop, VPN, company email, Slack, GitHub, Jira, 1Password.
Install IDE, Docker, Node/Python toolchain per dev-setup.md.
Run hello-world repo. Submit first PR on internal docs.
Day 3-4: Introductions & Context
15-min 1:1s with team members, manager, product owner, design lead.
Read company handbook sections: culture, engineering principles, incident response.
Review architecture docs: system diagram, API catalogue, deployment pipeline.
Day 5: First Task
Fix a labelled good-first-issue bug with guidance from buddy.
End-of-week checkpoint with manager: verify toolchain works, review team familiarity, confirm buddy assignment.
Week 1 Manager Review
Topics: access complete? Buddy assigned? Comfort level reading codebase? Any blockers.
WEEK 2 — IMMERSION
Read: on-call runbook, code review guidelines, style guide, testing pyramid doc.
Shadow one code review. Watch one deploy cycle.
Write one unit test for an existing module. Push branch. Open draft PR.
Attend sprint planning, standups, retro as observer.
Complete security awareness training (1 hour, mandatory).
End-of-week checkpoint with buddy: review draft PR together, discuss team rituals.
MONTH 1 — FIRST SHIP
Weeks 3-4
Own a small ticket (2-3 day effort) end-to-end: spec, implementation, tests, deploy.
Write or update documentation for the feature.
Give one code review to a peer.
Attend one incident review as listener.
Complete mandatory HR training: ethics, harassment, data privacy (self-paced, 2 hours total).
Month 1 Manager Review
Topics: ticket delivery quality, integration with team, code review feedback from peers, documentation habits, open questions about processes. Set goals for month 2.
MONTH 2-3 — INDEPENDENT CONTRIBUTOR
Month 2
Take a medium ticket (1 week) with minimal supervision.
Present a 5-min lightning talk at team demo (your feature, a tool you like, or a bug you fixed).
Pair-program with 2 different team members on their areas.
Read: production runbook — understand alerts, dashboards, escalation paths.
Shadow on-call rotation one day with senior engineer.
Month 3
Take a cross-team ticket or work with design/product on a new feature.
Write a post-mortem draft for a simulated incident.
Participate in sprint planning — estimate own tickets.
Option to join interview loop as observer.
Month 3 Manager Review
Topics: autonomy level, code quality trends, team collaboration, growth areas, promotion trajectory discussion, goals for next quarter.
CHECKPOINT SUMMARY
Week 1 — tools, team intros, first PR
Month 1 — first shipped ticket, code review given
Month 3 — independent work, cross-team exposure, trajectory plan
