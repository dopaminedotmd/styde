## 1. Brand Overview

**Brand Name:** Stryde
**Tagline:** *Move faster. Build better.*
**Sector:** B2B SaaS — Developer Productivity & CI/CD Platform
**Brand Archetype:** The Navigator / The Sage
**Core Promise:** Stryde transforms fragmented engineering pipelines into a single, intelligent velocity platform — so teams ship with confidence, not chaos.

### Brand Personality

| Trait | Description |
|---|---|
| **Confident** | We know our craft. We speak with authority rooted in deep engineering empathy. |
| **Precise** | Every pixel, every word, every interaction is intentional. |
| **Energetic** | Momentum matters. Our brand feels like forward motion. |
| **Approachable** | Complexity doesn't require coldness. We're warm, direct, and human. |
| **Optimistic** | We believe better tooling makes better teams — and we're here to build it. |

### Guiding Design Principles

1. **Motion implies progress.** Gradients, asymmetric layouts, and directional cues always point forward-right.
2. **Less chrome, more content.** Interfaces and marketing surfaces prioritize clarity over decoration.
3. **Depth without weight.** Layering, glassmorphism accents, and subtle shadows create dimension without density.
4. **Accessible by default.** All palettes, type scales, and interactive states meet **WCAG 2.2 Level AA** minimums.

### 2.4 Gradient System

Gradients are a signature element of the Stryde visual language. They always flow **left-bottom to right-top** (southwest → northeast) to imply upward-right momentum.

| Name | Stops | Use Case |
|---|---|---|
| **Brand Gradient** | `#2D1B69` → `#5B3FE8` → `#8B75F5` | Hero sections, mastheads, primary CTAs |
| **Accent Pulse** | `#5B3FE8` → `#C8FF00` | Feature callouts, progress indicators, loading states |
| **Dark Flow** | `#1A1040` → `#0D0D12` | Dark mode backgrounds, dashboard chrome |
| **Glass Overlay** | `rgba(91,63,232,0.08)` → `rgba(200,255,0,0.04)` | Glassmorphism cards, modal backdrops |

### 2.5 Usage Rules

- **Primary 500 (`#5B3FE8`)** must never drop below 15% of total color surface area in any branded composition.
- **Accent Lime (`#C8FF00`)** is reserved for interactive affordance. Never use it for body text, large background fills, or passive decoration.
- All text/background combinations must pass **4.5:1 contrast ratio** (AA). Exceptions: large text (≥18px bold or ≥24px regular) requires 3:1.
- Never tint or shade the Accent palette — use the tokens as-is. Primary and Neutral colors may be blended via opacity.

### 2.6 Dark Mode

Dark mode uses primary/neutral tokens natively — no separate palette. Backgrounds invert to `Primary 900` and `Neutral 900`. Text inverts to `Neutral 100` and `White`. Accent colors remain identical; their saturation naturally provides sufficient contrast against dark surfaces.

### 3.1 Typeface Selection

| Role | Typeface | Category | Licensing |
|---|---|---|---|

| **Body Text** | **Inter** | Neo-Grotesque Sans-Serif | OFL (Google Fonts) |

### 3.2 Font Stacks

```css
/* Headings */
--stryde-font-heading: 'Space Grotesk', 'Inter', system-ui, -apple-system, 'Segoe UI', Roboto, sans-serif;

/* Body */
--stryde-font-body: 'Inter', system-ui, -apple-system, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif;

/* Code / Monospace */
--stryde-font-mono: 'JetBrains Mono', 'Fira Code', 'Cascadia Code', 'Source Code Pro', 'Consolas', 'Courier New', monospace;
```

### 3.3 Type Scale

Based on a **major-third (1.25)** modular scale anchored at 16px root.

| Token | Size | Line-Height | Weight | Use |
|---|---|---|---|---|
| `text-display-xl` | 64px (4rem) | 1.1 | 700 | Marketing hero headlines |
| `text-display` | 48px (3rem) | 1.15 | 700 | Page titles, feature headers |
| `text-heading-xl` | 36px (2.25rem) | 1.2 | 700 | Section headers (H1) |
| `text-heading-lg` | 28px (1.75rem) | 1.25 | 600 | Subsection headers (H2) |
| `text-heading` | 22px (1.375rem) | 1.3 | 600 | Card headers, H3 |
| `text-heading-sm` | 18px (1.125rem) | 1.35 | 600 | Subheadings, H4 |
| `text-body-lg` | 18px (1.125rem) | 1.6 | 400 | Lead paragraphs, feature descriptions |
| `text-body` | 16px (1rem) | 1.6 | 400 | Body text, default paragraph |
| `text-body-sm` | 14px (0.875rem) | 1.55 | 400 | Secondary text, metadata, captions |
| `text-caption` | 12px (0.75rem) | 1.5 | 500 | Labels, badges, footnotes |
| `text-code` | 14px (0.875rem) | 1.6 | 400 | Inline code, code blocks |

### 3.4 Typography Rules

- **Headings: Space Grotesk only.** Never use Inter for H1–H4.
- **Letter-spacing:** Add `0.02em` to `text-caption` and `text-code`. Add `-0.01em` to `text-display-xl` and `text-display`.
- **Maximum line length:** 72 characters for body text. 52 characters for headings.
- **Heading hierarchy:** Never skip levels (H1 → H3 without H2 is prohibited).
- **Code in body:** Use JetBrains Mono within `<code>` or backtick spans. Inline code gets `background: --stryde-neutral-100`, `padding: 0.15em 0.4em`, `border-radius: 4px`.
- **Weight restrictions:** Space Grotesk is available in 300, 400, 500, 600, 700. Never use faux bold/italic.

### 3.5 Responsive Adjustments

| Breakpoint | Scale Adjustment |
|---|---|
| ≥ 1440px | Full scale as defined |
| 1024–1439px | `text-display-xl` → 56px; `text-display` → 40px |
| 768–1023px | `text-display-xl` → 44px; `text-display` → 34px |
| 480–767px | `text-display-xl` → 36px; `text-display` → 28px; all headings drop one level |
| < 480px | `text-display-xl` → 30px; `text-display` → 24px; body remains 16px minimum |

## 4. Logo & Wordmark Usage

### 4.1 Logo Construction

The Stryde logo consists of two inseparable parts:

1. **The Symbol (Icon Mark):** A stylized forward slash `/` formed by an ascending arc that curves into a sharp, rightward point. The arc is built from three gradient segments representing *source → build → deploy*. The symbol may *never* be used as the sole logo without the wordmark, except as a favicon or app icon.

2. **The Wordmark:** The word "Stryde" set in **Space Grotesk Bold** with custom letter-spacing (`0.05em`). The "y" descender is shortened by 15% to improve baseline alignment with adjacent characters.

### 4.2 Logo Variants

| Variant | Configuration | Background Requirement |
|---|---|---|
| **Primary (Horizontal)** | Symbol left, wordmark right | White or `Neutral 100` |
| **Primary (Stacked)** | Symbol centered above wordmark | White or `Neutral 100` |
| **Reversed (Horizontal)** | Symbol left, wordmark right — all white | `Primary 900` or `Primary 700` |
| **Reversed (Stacked)** | Symbol above, wordmark below — all white | `Primary 900` or `Primary 700` |
| **Monochrome** | Symbol + wordmark in `Neutral 900` | Light backgrounds only |
| **Symbol Only** | Isolated icon mark | Favicon, app icon, social avatar only |

### 4.3 Clear Space & Minimum Size

- **Clear space:** Equal to the height of the "S" in the wordmark on all four sides of the logo lockup.
- **Minimum width for print:** 32mm (horizontal), 24mm (stacked).
- **Minimum width for digital:** 120px (horizontal), 96px (stacked).
- **Minimum favicon/app icon:** 16×16px (symbol must remain recognizable at this size — simplified line weights apply below 32px).

### 4.4 Usage Prohibitions

| ❌ Never... |
|---|
| Stretch, squash, or distort the logo in any dimension |
| Recolor the symbol or wordmark outside of the approved variants |
| Apply drop shadows, glows, or effects to the logo |
| Place the logo on busy photographic backgrounds without a protective clear zone |
| Rotate the logo |
| Use the wordmark without the symbol (except in plain-text contexts like legal documents or emails) |
| Enclose the logo in a shape (circle, square, pill) unless it's the symbol-only favicon variant |
| Change the typeface of the wordmark |
| Outline the logo |

### 4.5 Co-Branding

When Stryde appears alongside partner logos:
- Use a vertical divider (`Neutral 300`, 1px, 24px height) between logos.
- Stryde logo size must match partner logo optical size (not absolute dimensions — match visual weight).
- Minimum 2x clear space between any partner logo and Stryde.

## 5. Iconography

### 5.1 Icon System

Stryde uses a custom icon set called **Stryde Icons**, built on a **24×24px grid** with a **2px stroke weight**.

| Property | Specification |
|---|---|
| **Grid size** | 24 × 24 dp |
| **Stroke weight** | 2px |
| **Stroke cap** | Round |
| **Stroke join** | Round |
| **Corner radius** | 2px (internal corners), 0px (external corners) |
| **Padding** | 2px internal safe zone |
| **Style** | Linear, with solid fills for selected/active states |

### 5.2 Icon Categories

| Category | Visual Direction | Examples |
|---|---|---|
| **Navigation** | Geometric, directional cues | Home, Dashboard, Settings, Search |
| **Actions** | Expressive strokes, motion arcs | Add, Edit, Delete, Share, Download |
| **Pipeline** | Git-branch motifs, flow arrows | Build, Deploy, Test, Merge, Branch |
| **Status** | Filled states with color coding | Success (Lime), Warning (Warm), Error (Red), Info (Cool) |
| **Objects** | Recognizable silhouettes | Server, Database, Cloud, Shield, Key |
| **Files & Data** | Document outlines, data grids | File, Folder, Table, Chart, JSON |

### 5.3 Usage Rules

- **Sizing:** Use multiples of 4: 16px (compact), 20px (inline), 24px (default), 32px (large UI), 48px (feature illustrations).
- **Inline with text:** Vertically align to the text baseline, not center. Use `vertical-align: text-bottom`.
- **Color inheritance:** Icons inherit their parent's text color unless explicitly set to a status color.
- **Interaction states:** Icons in buttons or links respond to parent state — hover, focus, active, disabled — without independent color changes.
- **Never mix:** Do not combine Stryde Icons with third-party icon libraries in the same viewport unless there's a clear functional distinction (e.g., Stryde Icons for UI, third-party for social media badges).

### 5.4 Empty State Illustrations

Empty states use enlarged icons (64–96px) paired with a soft gradient background (`Primary 100` → `Neutral 100`). The icon sits inside a rounded square or circle with `border-radius: 16px`. Accompanying text uses `text-body-lg` in `Neutral 500` with a `text-heading-sm` title.

## 6. Photography & Image Style

### 6.1 Photographic Direction

Stryde photography conveys **focus, motion, and human collaboration** in technical environments.

| Element | Direction |
|---|---|
| **Subject** | Developers, engineers, and teams at work — candid, not posed. Equipment and interfaces visible but not dominant. |
| **Composition** | Rule of thirds. Subject often placed on the left third, facing or moving right. Negative space on the right for text overlay. |
| **Lighting** | High-key natural light with cool color temperature (5500K–6500K). Rim lighting from top-right. Minimal harsh shadows. |
| **Depth of Field** | Shallow (f/1.4–f/2.8). Background softly blurred to isolate subjects. |
| **Color Grading** | Desaturated shadows (cool bias), neutral midtones, slightly warm highlights. Overall cool-leaning but not sterile. |
| **Aspect Ratios** | 16:9 (hero), 3:2 (cards), 1:1 (avatars), 4:3 (blog thumbnails) |

### 6.2 Photo Treatments

All photography passes through a Stryde color grade:

1. **Desaturate reds and yellows** by 15–25% (removes skin-tone warmth that clashes with the cool palette).
2. **Lift blue/purple midtones** by 8–12% (harmonizes with `Primary 500`).
3. **Apply subtle vignette** (20–30% opacity, soft radial gradient) to focus attention on the subject.
4. **Overlay gradient** on hero images: left edge `rgba(26,16,64,0.4)` fading to transparent at center.

### 6.3 Image Selection Criteria

| ✅ Use | ❌ Avoid |
|---|---|
| Authentic workspace environments | Obvious stock photography tropes (handshakes, generic office) |
| Screens showing real interfaces | Screens showing competitors' products |
| Diverse teams in collaboration | Single-person-at-desk isolation shots |
| Motion blur suggesting activity | Static, sterile compositions |
| Natural light sources | Harsh fluorescent lighting |
| Clean, modern hardware | Cluttered desks or outdated equipment |

### 6.4 Illustration vs. Photography

- **Marketing pages:** 60% photography, 40% illustration
- **Product UI:** 10% photography (avatars only), 90% illustration/iconography
- **Blog & content:** 70% photography, 30% data visualization
- **Documentation:** 0% photography, 100% diagrams/icons/screenshots

## 7. Illustration & Graphic Motifs

### 7.1 Illustration Style

Stryde illustrations follow a **geometric vector** aesthetic:

| Property | Specification |
|---|---|
| **Style** | Flat vector with gradient accents |
| **Shapes** | Geometric primitives (circles, rounded rectangles, triangles) composed into scenes |
| **Lines** | 2px stroke, rounded caps — consistent with icon system |
| **Color** | Palette-restricted: `Primary 500`, `Primary 300`, `Primary 100`, `Accent`, `Neutral 300` |
| **Gradients** | 2-stop Brand Gradient on focal elements only |
| **Perspective** | Isometric or flat orthographic — never vanishing-point perspective |
| **Shadows** | Soft drop shadows only (`0px 8px 24px rgba(26,16,64,0.12)`), never cast shadows on ground planes |

### 7.2 Recurring Motifs

| Motif | Symbolism | Usage |
|---|---|---|
| **Forward Slash `/`** | Progress, code, path selection | Logo, dividers, decorative elements |
| **Ascending Arc** | Growth, deployment, momentum | Charts, pipeline diagrams, hero backgrounds |
| **Three-Dot Cluster** | Build → Test → Deploy pipeline | Feature icons, loading states, section breaks |
| **Hexagon Grid** | Engineering, systems, infrastructure | Background patterns, card decorations |
| **Data Nodes & Edges** | Connectivity, graph data, pipeline topology | Infographics, architecture diagrams |

### 7.3 Background Patterns

- **Dot Grid:** `Neutral 300` dots at 1px radius, 24px spacing, 15% opacity — used on light backgrounds for subtle texture.
- **Hex Grid:** `Primary 300` hexagons at 48px, stroked at 1px, 8% opacity — used on `Primary 100` backgrounds.
- **Gradient Mesh:** Blurred (40px) circles of `Primary 500` and `Accent` at 5–10% opacity — hero section backdrops.

- Labels: `text-body-sm`, `Neutral 700`, above input with 8px gap

## 10. Brand Voice & Tone

### 10.1 Voice Attributes

| Attribute | Description |
|---|---|
| **Direct** | We say what we mean without hedging. Short sentences. Active voice. |
| **Knowledgeable** | We reference real engineering concepts with accuracy. No hand-waving. |
| **Encouraging** | We meet teams where they are. We never shame legacy tooling or current practices. |
| **Witty (Controlled)** | Occasional dry humor. Puns are fine in social — never in docs or error messages. |
| **Inclusive** | Gender-neutral language. "Folks" or "teams" over "guys." Accessible by design. |

### 10.2 Tone by Context

| Context | Tone | Example |
|---|---|---|
| **Marketing Headlines** | Bold, aspirational | *"Your pipeline shouldn't be the bottleneck. It should be the engine."* |
| **Product UI** | Concise, helpful | *"No pipelines yet. Import one in 3 clicks."* |
| **Documentation** | Precise, instructive | *"Stryde resolves dependencies before executing any job in the graph."* |
| **Error Messages** | Clear, blame-free, actionable | *"We couldn't connect to your repo. Check your access token and try again."* |
| **Blog Posts** | Insightful, conversational | *"Here's what 10,000 CI minutes taught us about flaky tests — and how to fix them."* |
| **Social Media** | Playful, community-minded | *"`git push —force`? We trust you... but Stryde's got your back just in case."* |
| **Email / Newsletter** | Personal, value-first | *"Hey {{name}}, your team shipped 40% faster this month. Here's the breakdown."* |
| **Support Responses** | Empathetic, efficient | *"That's frustrating — let's get this sorted. Can you share your pipeline ID?"* |

### 10.3 Writing Rules

1. **Active voice always.** *"Stryde deploys your app"* — never *"Your app is deployed by Stryde."*
2. **Sentence case for UI.** Buttons, labels, headings, and menu items use sentence case: *"Create pipeline"* not *"Create Pipeline."*
3. **No exclamation marks in product UI.** Exclamation marks are allowed in marketing headlines and social media (maximum one per piece).
4. **Oxford comma required.** *"Build, test, and deploy"* — not *"Build, test and deploy."*
5. **Numbers under 10 spelled out** in prose — digits for UI counts and metrics.
6. **"Stryde" is singular.** *"Stryde is a CI/CD platform"* — not *"Stryde are..."*
7. **Avoid "easy," "simple," "just."** These words dismiss user effort. Show, don't claim.
8. **Code references:** Backtick-wrap command names, file paths, and env vars in prose. Use bold for UI element names.

### 10.4 Glossary

| Term | Definition |
|---|---|
| **Pipeline** | A sequence of automated jobs triggered by events |
| **Run** | A single execution of a pipeline |
| **Job** | An individual unit of work within a run |
| **Workflow** | A pipeline template reusable across projects |
| **Trigger** | An event that starts a pipeline (push, PR, schedule, manual) |
| **Artifact** | Any file produced by a job and persisted for downstream jobs |
| **Cache** | Stored dependencies to speed up subsequent runs |
| **Runner** | The execution environment that processes jobs |

- Left third: Headline in `text-display-xl` (Space Grotesk Bold, `Neutral 900`) on `White` background. Subhead in `text-body-lg` (`Neutral 700`). Primary CTA button (`Primary 500` bg, `White` text, `radius-md`).
- Right two-thirds: Abstract 3D isometric illustration of a pipeline graph with `Primary 500` nodes connected by `Accent` edges. Gradient mesh background (`Primary 100` → transparent). The `/` symbol motif subtly woven into node connections.

**Dashboard (Product UI):**
- Dark mode by default (`Primary 900` background).
- Left sidebar: 240px, `Primary 700` with glassmorphism surface (blur 16px, 80% opacity).
- Top nav: 56px height, same glass treatment.
- Cards in content area: `Neutral 900` at 0.6 opacity, `radius-md`, subtle `Neutral 700` border.
- Pipeline status indicators use `Accent` for success, `Accent Warm` for failure, `Accent Cool` for in-progress.
- Code snippets in cards use JetBrains Mono on `Neutral 900` solid background with `Primary 500` syntax highlighting accents.
