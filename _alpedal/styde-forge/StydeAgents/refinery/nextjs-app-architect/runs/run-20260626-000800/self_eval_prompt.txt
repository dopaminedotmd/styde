## 1. Architecture Decisions & Rationale

| Decision | Choice | Rationale |
|---|---|---|
| **Router** | App Router (not Pages) | React Server Components, streaming, nested layouts, colocation |
| **Rendering** | Server-first, client islands | SEO, bundle size, performance baseline |
| **State management** | URL + Server Context + React Query (client) | Avoid dual-sourcing truth; cache-then-network for mutations |
| **Auth strategy** | Edge middleware + JWT (httpOnly cookie) + NextAuth v5 | Latency: middleware runs at edge before cache lookup |
| **Styling** | Tailwind CSS + CSS Modules for colocated styles | Utility-first with escape hatch for complex component styles |
| **Database** | Drizzle ORM + PostgreSQL (server-only) | Type-safe, lightweight, works at edge with pooled connections |
| **Validation** | Zod on server boundaries (actions, handlers, middleware) | Single source of truth for API and form validation |
| **Caching** | `fetch` cache + `unstable_cache` + ISR `revalidate` | Fine-grained per-route control |
| **Error tracking** | `error.tsx` boundaries + structured logging (Pino) | Server errors caught by boundaries, client errors by React Error Boundaries |
| **Monorepo** | No — single Next.js app | Simplicity; extract shared packages only when needed |

### Key Architectural Principles

1. **Server Components by default** — flip to `'use client'` only when interactivity or browser APIs are required.
2. **Composition over configuration** — layouts compose via `children`; route groups compose via folder naming.
3. **Streaming over blocking** — every data-dependent route has a `loading.tsx`; long queries use `<Suspense>` for selective streaming.
4. **Middleware is the only gate** — no `getServerSideProps`-style auth checks scattered across pages; auth lives in `middleware.ts` and is enforced by layout-level server checks as a second layer.
5. **Route handlers as BFF** — `/api/*` handlers are Backend-For-Frontend, aggregating internal services.

### Group: `(dashboard)` — Authenticated workspace

```tsx
// src/app/(dashboard)/layout.tsx
import { redirect } from 'next/navigation';
import { getServerSession } from '@/lib/auth/session';
import { Sidebar } from '@/components/layout/sidebar';
import { Topbar } from '@/components/layout/topbar';

export default async function DashboardLayout({ children }: { children: React.ReactNode }) {
  const session = await getServerSession();
  if (!session) redirect('/login?callbackUrl=/dashboard');

  return (
    <div className="flex h-screen overflow-hidden">
      <Sidebar user={session.user} />
      <div className="flex flex-1 flex-col overflow-hidden">
        <Topbar user={session.user} />
        <main className="flex-1 overflow-y-auto p-6">{children}</main>
      </div>
    </div>
  );
}
```

### Why route groups over `layout` nesting?

- **URL cleanliness**: `/dashboard/projects` stays clean, no `/app/dashboard/projects`.
- **Independent layouts**: Each group gets its own shell. Group-switching is seamless.
- **Code organization**: Colocation by concern — marketing, auth, workspace live in clearly named folders.

## 5. Loading States & Streaming

Next.js wraps `page.tsx` and `layout.tsx` in `<Suspense>` automatically, using `loading.tsx` as the fallback. For more granular streaming, use **manual `<Suspense>` boundaries**.

### Route-level loading (automatic streaming)

```tsx

      <Skeleton className="h-8 w-48" />               {/* Title */}
      <div className="grid grid-cols-3 gap-4">
        {Array.from({ length: 6 }).map((_, i) => (
          <Skeleton key={i} className="h-40 rounded-lg" />  {/* Project cards */}
        ))}
      </div>
    </div>
  );
}
```

### Selective streaming with `<Suspense>`

```tsx
// src/app/(dashboard)/projects/[projectId]/page.tsx
import { Suspense } from 'react';
import { ProjectHeader } from './project-header';
import { ProjectDetails } from './project-details';
import { DeferredAnalytics } from '@/components/shared/streaming/deferred-analytics';
import { Skeleton } from '@/components/ui/skeleton';

export default async function ProjectPage({ params }: { params: { projectId: string } }) {
  return (
    <div className="space-y-6">
      {/* Fetches fast — rendered immediately */}
      <ProjectHeader projectId={params.projectId} />

      {/* Fetches medium — skeleton shown briefly */}
      <Suspense fallback={<Skeleton className="h-64 rounded-lg" />}>
        <ProjectDetails projectId={params.projectId} />
      </Suspense>

      {/* Fetches slow (analytics query) — streams in last */}
      <Suspense fallback={<Skeleton className="h-48 rounded-lg" />}>
        <DeferredAnalytics projectId={params.projectId} />
      </Suspense>
    </div>
  );
}
```

```tsx
// src/components/shared/streaming/deferred-analytics.tsx
import { db } from '@/lib/db';
import { analytics } from '@/lib/db/schema';
import { eq, sql } from 'drizzle-orm';

interface Props {
  projectId: string;
}

export async function DeferredAnalytics({ projectId }: Props) {
  // This query might take 2-3 seconds — it won't block the page shell
  const [data] = await db
    .select({
      totalViews: sql<number>`count(*)`,
      avgDuration: sql<number>`avg(duration)`,
    })
    .from(analytics)
    .where(eq(analytics.projectId, projectId));

  return (
    <div className="grid grid-cols-2 gap-4">
      <StatCard label="Total Views" value={data.totalViews} />
      <StatCard label="Avg. Duration" value={`${Math.round(data.avgDuration)}s`} />
    </div>
  );
}
```

### Streaming strategy

| Content Type | Strategy | Fallback |
|---|---|---|
| Page shell (layout, nav) | Immediate render | None |
| Primary content | Route-level `loading.tsx` | Full-page skeleton |
| Fast sub-components | Inline fetch (no Suspense) | None needed |
| Medium sub-components | `<Suspense>` with skeleton | Per-component skeleton |
| Slow analytics / charts | `<Suspense>` with lowest priority | Placeholder card |

## 8. Server Components with Streaming

### Pattern: Single async Server Component

```tsx
// src/app/(marketing)/blog/[slug]/page.tsx
import { db } from '@/lib/db';
import { posts } from '@/lib/db/schema';
import { eq } from 'drizzle-orm';
import { notFound } from 'next/navigation';
import { Metadata } from 'next';

interface Props {
  params: { slug: string };
}

// ISR: revalidate every 60 seconds (or on-demand via revalidateTag)
export const revalidate = 60;

export async function generateMetadata({ params }: Props): Promise<Metadata> {
  const post = await db.select({ title: posts.title, excerpt: posts.excerpt })
    .from(posts)
    .where(eq(posts.slug, params.slug))
    .limit(1);

  return {
    title: post[0]?.title ?? 'Post not found',
    description: post[0]?.excerpt,
  };
}

// generateStaticParams for SSG pre-rendering of known slugs
export async function generateStaticParams() {
  const allPosts = await db.select({ slug: posts.slug }).from(posts);

      <h1>{post.title}</h1>
      <time dateTime={post.publishedAt.toISOString()}>
```

### Custom login handler

```ts
// src/app/api/auth/login/route.ts
import { NextRequest, NextResponse } from 'next/server';
import { z } from 'zod';
import { validateBody } from '@/lib/api/validate';
import { db } from '@/lib/db';
import { users } from '@/lib/db/schema';
import { eq } from 'drizzle-orm';
import { compare } from 'bcrypt-ts';
import { signJwt } from '@/lib/auth/jwt';

const loginSchema = z.object({
  email: z.string().email(),
  password: z.string().min(8),
});

export async function POST(request: NextRequest) {
  // Validate body
  const parsed = await validateBody(request, loginSchema);
  if (parsed instanceof NextResponse) return parsed; // 400 error response

  const { email, password } = parsed;

  // Lookup user
  const [user] = await db.select().from(users).where(eq(users.email, email));
  if (!user) {
    return NextResponse.json(
      { error: 'Invalid credentials' },
      { status: 401 }
    );
  }

  // Verify password
  const valid = await compare(password, user.passwordHash);
  if (!valid) {
    return NextResponse.json(
      { error: 'Invalid credentials' },
      { status: 401 }
    );
  }

  // Issue JWT
  const token = await signJwt({ sub: user.id, email: user.email });

  const response = NextResponse.json({ success: true });
  response.cookies.set('auth-token', token, {
    httpOnly: true,
    secure: process.env.NODE_ENV === 'production',
    sameSite: 'lax',
    path: '/',
    maxAge: 60 * 60 * 24 * 7, // 7 days
  });

  return response;
}
```

```ts
      // Handle cancellation
```

### Validation helper

```ts
// src/lib/api/validate.ts
import { NextRequest, NextResponse } from 'next/server';
import { ZodSchema } from 'zod';

export async function validateBody<T>(
  request: NextRequest,
  schema: ZodSchema<T>
): Promise<T | NextResponse> {
  try {
    const body = await request.json();
    const result = schema.safeParse(body);

    if (!result.success) {
      return NextResponse.json(
        {
          error: 'Validation failed',
          details: result.error.flatten().fieldErrors,
        },
        { status: 400 }
      );
    }

    return result.data;
  } catch {
    return NextResponse.json(
      { error: 'Invalid JSON body' },
      { status: 400 }
    );
  }
}
```

## 10. Middleware for Auth

Middleware runs **at the Edge** before cache lookup. It's the lowest-latency place to gate access.

```ts
// src/middleware.ts
import { NextResponse } from 'next/server';
import type { NextRequest } from 'next/server';
import { jwtVerify } from 'jose';

const JWT_SECRET = new TextEncoder().encode(process.env.JWT_SECRET!);

// Config: which paths middleware applies to
export const config = {
  matcher: [
    '/dashboard/:path*',
    '/api/protected/:path*',
    '/login',
    '/register',
  ],
};

export async function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl;

  // --- Auth pages: redirect if already authenticated ---
  if (pathname.startsWith('/login') || pathname.startsWith('/register')) {
    const token = request.cookies.get('auth-token')?.value;
    if (token) {
      try {
        await jwtVerify(token, JWT_SECRET);
        return NextResponse.redirect(new URL('/dashboard', request.url));
      } catch {
        // Invalid token — stay on login page
      }
    }
    return NextResponse.next();
  }

  // --- Protected routes: enforce authentication ---
  const token = request.cookies.get('auth-token')?.value;

  if (!token) {
    // API routes: return 401
    if (pathname.startsWith('/api/')) {
      return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });
    }
    // Page routes: redirect to login with callback URL
    const loginUrl = new URL('/login', request.url);
    loginUrl.searchParams.set('callbackUrl', pathname);
    return NextResponse.redirect(loginUrl);
  }

  try {
    const { payload } = await jwtVerify(token, JWT_SECRET);

    // Attach user info to headers for downstream consumption
    const requestHeaders = new Headers(request.headers);
    requestHeaders.set('x-user-id', payload.sub as string);
    requestHeaders.set('x-user-email', payload.email as string);

    return NextResponse.next({
      request: { headers: requestHeaders },
    });
  } catch {
    // Token invalid or expired
    const response = NextResponse.redirect(new URL('/login', request.url));
    response.cookies.delete('auth-token');
    return response;
  }
}

// Edge-compatible JWT (jose) — works in Edge runtime, unlike jsonwebtoken
```

### Server-side session retrieval (after middleware)

```ts
// src/lib/auth/session.ts
import { headers } from 'next/headers';

export interface Session {
  userId: string;
  email: string;
}

export async function getServerSession(): Promise<Session | null> {
  const headersList = headers();
  const userId = headersList.get('x-user-id');
  const email = headersList.get('x-user-email');

  if (!userId || !email) return null;

  return { userId, email };
}
```

### Auth gating in layout (defense-in-depth)

```tsx
// src/app/(dashboard)/layout.tsx (excerpt)
export default async function DashboardLayout({ children }: { children: React.ReactNode }) {
  const session = await getServerSession();
  if (!session) redirect('/login?callbackUrl=/dashboard');
  // ...
}
```

**Why both middleware and layout?**
- **Middleware** runs at edge, blocks before any rendering. Fast deny.
- **Layout** runs on Node.js, can access database for richer auth checks (roles, permissions, feature flags).
- Both layers together prevent leaking protected content even if middleware is misconfigured.

### fetch-level cache control

```tsx
// Per-fetch ISR timing
```

### On-demand revalidation (API route)

```ts
// src/app/api/revalidate/route.ts
import { revalidateTag, revalidatePath } from 'next/cache';
import { NextRequest, NextResponse } from 'next/server';

// Protected by a secret token (set in env)
export async function POST(request: NextRequest) {
  const body = await request.json();
  const { secret, tag, path } = body;

  if (secret !== process.env.REVALIDATION_SECRET) {
    return NextResponse.json({ error: 'Invalid secret' }, { status: 401 });
  }

  if (tag) {
    revalidateTag(tag);
    return NextResponse.json({ revalidated: true, tag });
  }

  if (path) {
    revalidatePath(path);
    return NextResponse.json({ revalidated: true, path });
  }

  return NextResponse.json({ error: 'Provide tag or path' }, { status: 400 });
}
```

### ISR strategy matrix

| Route | Strategy | Revalidate | Rationale |
|---|---|---|---|
| `/` (homepage) | ISR | 300s | Changes infrequently; fast TTFB |
| `/blog` (list) | ISR + tag | 300s | Revalidate on new post publish |
| `/blog/[slug]` | ISR | 3600s | Content rarely changes |
| `/pricing` | SSG | ∞ (build time) | Static content |
| `/dashboard` | SSR (dynamic) | 0 | Personalized content |
| `/dashboard/projects/[id]` | SSR (dynamic) | 0 | Real-time data |
| `/api/*` | Dynamic | N/A | API endpoints always fresh |

## 12. Data Fetching Patterns

### Server-only data layer

```ts
// src/lib/db/queries.ts
import { cache } from 'react';
import { db } from './index';
import { projects, type Project } from './schema';
import { eq } from 'drizzle-orm';
import { CACHE_TAGS } from '@/lib/cache/keys';

// React cache() deduplicates multiple calls in the same render pass
export const getProject = cache(async (id: string): Promise<Project | null> => {
  const [project] = await db
    .select()
    .from(projects)
    .where(eq(projects.id, id))
    .limit(1);

  return project ?? null;
});

export const getProjects = cache(async (userId: string): Promise<Project[]> => {
  return db
    .select()
    .from(projects)
    .where(eq(projects.ownerId, userId))
    .orderBy(projects.updatedAt);
});
```

### Fetch wrapper with cache tagging

```ts
// src/lib/fetch.ts
export async function cachedFetch<T>(
  url: string,
  tags: string[],
  revalidate?: number
): Promise<T> {
  const res = await fetch(url, {
    next: {
      tags,
      ...(revalidate ? { revalidate } : {}),
    },
  });

  if (!res.ok) {
    throw new Error(`Fetch failed: ${res.status} ${res.statusText}`);
  }

  return res.json();
}
```

### Server Actions for mutations

```tsx
// src/app/(dashboard)/projects/new/actions.ts
'use server';

import { revalidateTag } from 'next/cache';
import { redirect } from 'next/navigation';
import { z } from 'zod';
import { db } from '@/lib/db';
import { projects } from '@/lib/db/schema';
import { getServerSession } from '@/lib/auth/session';
import { CACHE_TAGS } from '@/lib/cache/keys';

const createProjectSchema = z.object({
  name: z.string().min(1).max(100),
  description: z.string().max(500).optional(),
});

export async function createProject(formData: FormData) {
  const session = await getServerSession();
  if (!session) throw new Error('Unauthorized');

  const parsed = createProjectSchema.safeParse({
    name: formData.get('name'),
    description: formData.get('description'),
  });

  if (!parsed.success) {
    return { errors: parsed.error.flatten().fieldErrors };
  }

  const [project] = await db
    .insert(projects)
    .values({
      name: parsed.data.name,
      description: parsed.data.description ?? null,
      ownerId: session.userId,
    })
    .returning();

  revalidateTag(CACHE_TAGS.PROJECTS_LIST);
  redirect(`/dashboard/projects/${project.id}`);
}
```

## Summary of Key Patterns

| Concern | Solution |
|---|---|
| **Auth gate** | Edge `middleware.ts` + layout `getServerSession()` |
| **Layout per section** | Route groups `(marketing)`, `(auth)`, `(dashboard)` |
| **Streaming** | `loading.tsx` per route + manual `<Suspense>` for granularity |
| **Errors** | `error.tsx` per route segment + `global-error.tsx` for root |
| **404** | `not-found.tsx` per segment + `notFound()` function |
| **API layer** | `route.ts` handlers with Zod validation |
| **Caching** | `revalidate` export + `revalidateTag()` on mutations |
| **Data fetching** | React `cache()` + Drizzle ORM, colocated in `lib/db/queries.ts` |
| **Mutations** | Server Actions with Zod, redirect, and cache revalidation |
| **Types** | Shared in `src/types/`, env validated at startup |
