text-wrap: balance

In the world of web typography, text-wrap: balance ensures that each line of a heading or short paragraph contains roughly the same number of characters. The browser intelligently wraps text so no single line is dramatically shorter than the others — perfect for headlines, callouts, and deck copy.

text-wrap: pretty

For running body text, text-wrap: pretty prevents orphans — those lonely single words dangling on the last line of a paragraph. The layout engine can pull a word from the previous line to ensure at least two words occupy the final line, producing a more polished, professional reading experience that rivals print typography.

text-wrap: wrap (default)

Without any special text-wrap strategy, this paragraph uses the browser's default greedy line-breaking algorithm. Notice how the last line might contain just one dangling word — an orphan — and line lengths may feel uneven compared to the balanced and pretty versions above. Compare all three to appreciate the difference.

initial-letter — Drop Caps

Typography has deep roots in manuscript traditions, where scribes would ornament the first letter of a chapter with elaborate flourishes. Today, CSS's initial-letter property brings this centuries-old technique to the web — cleanly, without hacks. Chapter I: The Nature of Type

Good typography is invisible. When type is set well, the reader forgets they are reading and simply falls into the text. The measure — line length — should be comfortable for the eye to track without losing its place. Traditionally, a column width of 45–75 characters per line is considered ideal for continuous reading. Multi-column layouts let us present more text in a given viewport while maintaining readable line lengths.

Chapter II: Variable Fonts

Variable fonts represent a paradigm shift in digital typography. Instead of shipping multiple static font files — regular, bold, italic, condensed, expanded — a single variable font file contains all those variations along continuous axes. Designers can interpolate seamlessly between weights (100–900), widths (75–125), slants, optical sizes, and custom axes defined by the type designer. This engine exposes four of those axes and wires them to live sliders.

Chapter III: Fluid Responsive Type

CSS clamp() enables type that scales smoothly between a minimum and maximum size based on the viewport width. Crucially, because the calculation uses rem units and viewport-relative units, it respects the user's browser zoom level and their chosen default font size. This is not merely "responsive" — it is fluid, adaptive, and accessible. The type scales continuously with no breakpoint jumps, while still honoring the reader's preferences.

Chapter IV: The Reading Experience

Every decision in this engine — from the font-size preference slider to the text-wrap strategy — serves one goal: a better reading experience. Users can enlarge text without breaking layout. Variable font axes adjust to taste and context. Balanced headlines feel crafted. Drop caps add character. Multi-column layouts respect the reader's screen real estate. Together, these features form a typographic system that is dynamic, respectful, and beautiful.
