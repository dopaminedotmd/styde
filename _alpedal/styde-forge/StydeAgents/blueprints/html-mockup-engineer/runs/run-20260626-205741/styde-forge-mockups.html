Blueprint
A directory under StydeAgents/blueprints/ containing three required files (BLUEPRINT.md, persona.md, config.yaml) that define an AI agent's purpose, personality, and configuration.
Refinery
The active training environment where agents are iterated through spawn-eval-improve loops. Agents stay here until their composite score reaches the production threshold.
Production
The deployment state for agents that have achieved composite score >= 85 (or 95 for General/Fas 0.5 tiers). Production agents are ready to receive real tasks.
Composite Score
The final evaluation score calculated as Self-Eval x 0.4 + Judge-Eval x 0.6. Self-Eval is the agent rating its own output; Judge-Eval is a fresh agent with no context rating the output independently.
Circuit Breaker
A safety system that halts pipeline operations after repeated failures or API errors to prevent runaway costs. Includes rate limiting, backoff timers, and automatic retry scheduling.
Tier Target
Each agent belongs to a tier (General, Fas 0.5, Fas 1-6). General and Fas 0.5 agents require composite >= 95 for production; all other tiers require composite >= 85.

⚡Pipeline Actions

Commands execute via python Core/forge.py. Each command wraps Hermes delegate_task calls.
Self-Eval
The agent rates its own output on five dimensions (accuracy, clarity, completeness, efficiency, usefulness), each 0-100. Provides the agent's internal quality estimate.
Judge-Eval
A fresh Hermes delegate_task with zero prior context independently rates the same output. This prevents the agent's familiarity with the blueprint from biasing the score. Weighted at 60% of composite.
Composite Score
The final score = Self-Eval x 0.4 + Judge-Eval x 0.6. Production threshold is 85 (or 95 for General/Fas 0.5 tiers).
Teacher Agent
A dedicated evaluator Hermes delegate_task that reads Self-Eval and Judge-Eval results, identifies the weakest dimension (lowest score), and proposes concrete improvements to the blueprint files.
Auto-Version
Each Teacher improvement bumps the version number in config.yaml. Versions track iterations through the spawn-eval-improve loop. Checkpoints save state before each change.
Fas 0.5
A design-specialist agent tier focused on visual mockups, HTML/CSS engineering, and interface design. Targets composite score of 95 for production promotion.

Caveman Ultra Mode
Default output mode that strips all markdown characters (#, **, `, -, >, ---) from prompts and agent output. Reduces token consumption by approximately 30-40%. Toggle via: python Core/forge.py caveman-toggle.
Circuit Breaker
Safety system that tracks consecutive failures per operation. After 3 failures, backs off for the specified interval. Max backoff caps the wait time. Resets after the reset interval of no failures.
Hermes Bridge
The integration layer that wraps Hermes delegate_task() calls. Each spawn creates a Hermes sub-agent. The bridge handles payload construction, response parsing, and error reporting.
Smart Cache
An LRU (Least Recently Used) cache that stores blueprint context (BLUEPRINT.md + persona.md + config.yaml) after first load. Subsequent evaluations reuse cached context for faster throughput.
Lock File (.forge.lock)
A filesystem-level lock that prevents multiple forge.py processes from running simultaneously. Written at process start, cleaned up on clean exit. If stale lock detected, recovery.py handles cleanup.
Auto-Versioning
Each Teacher improvement automatically bumps the version field in the blueprint's config.yaml. Versions are tracked in state.yaml alongside the change history. Named checkpoints can be created before risky changes.
Staleness Check
An automated scan that detects blueprints whose last pipeline activity exceeds the configured threshold (default 24 hours). Stale blueprints are flagged for re-evaluation or re-spawn.
