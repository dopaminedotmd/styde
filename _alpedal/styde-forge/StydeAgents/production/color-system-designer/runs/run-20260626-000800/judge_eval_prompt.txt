## 1. OKLCH Color Space Primer

### Why OKLCH?

OKLCH is the cylindrical form of the Oklab perceptual color space, designed by Björn Ottosson to fix the hue uniformity problems in CIELAB and CIELCH. It offers:

- **Perceptual uniformity:** Equal numeric changes produce equal perceived changes.
- **Hue linearity:** Blues don't shift to purple when adjusting lightness (a notorious CIELAB flaw).
- **Gamut awareness:** Chroma values map directly to sRGB/P3 gamut boundaries.
- **Human readable:** `oklch(60% 0.15 250)` — lightness, chroma, hue — is intuitive to author.

### Coordinate System

| Axis | Range | Description |
|------|-------|-------------|
| **L** (Lightness) | 0% – 100% | Perceptual lightness; 0 = black, 100 = white |
| **C** (Chroma) | 0 – 0.4+ | Color intensity/saturation; 0 = gray, higher = more vivid |
| **H** (Hue) | 0° – 360° | Hue angle on the color wheel |

### Key Hue Reference Points (OKLCH)

| Color | Hue Angle | Description |
|-------|-----------|-------------|
| Red | ~28° | Pure red anchor |
| Orange | ~60° | Warm orange |
| Yellow | ~110° | Pure yellow |
| Green | ~142° | Natural green |
| Teal | ~180° | Blue-green midpoint |
| Cyan | ~210° | Cyan |
| Blue | ~264° | Pure blue |
| Purple | ~302° | Violet/purple |
| Magenta | ~350° | Pink/magenta |

## 2. WCAG Contrast Ratio Functions

### The Formula

WCAG 2.x defines **relative luminance** and **contrast ratio** for sRGB colors:

```
// Step 1: Linearize sRGB channel
function linearize(channel: number): number {
  const c = channel / 255;
  return c <= 0.04045 ? c / 12.92 : ((c + 0.055) / 1.055) ** 2.4;
}

// Step 2: Compute relative luminance
function relativeLuminance(r: number, g: number, b: number): number {
  return 0.2126 * linearize(r) + 0.7152 * linearize(g) + 0.0722 * linearize(b);
}

// Step 3: Compute contrast ratio
function contrastRatio(lum1: number, lum2: number): number {
  const lighter = Math.max(lum1, lum2);
  const darker  = Math.min(lum1, lum2);
  return (lighter + 0.05) / (darker + 0.05);
}
```

### WCAG Conformance Levels

| Level | Normal Text (< 24px) | Large Text (≥ 24px, or ≥ 18.67px bold) | UI Components |
|-------|---------------------|----------------------------------------|---------------|
| **AA** | ≥ 4.5:1 | ≥ 3:1 | ≥ 3:1 |
| **AAA** | ≥ 7:1 | ≥ 4.5:1 | — |

### OKLCH-to-Contrast: The Bridge

Since WCAG contrast uses sRGB relative luminance, we must convert OKLCH → sRGB → relative luminance. The conversion path is:

```
OKLCH → Oklab → linear sRGB → gamma-corrected sRGB → relative luminance → contrast ratio
```

### Contrast-Ratio API (TypeScript)

```typescript
/**
 * Compute WCAG 2.1 contrast ratio between two OKLCH colors.
 */
function contrastOklch(
  l1: number, c1: number, h1: number,
  l2: number, c2: number, h2: number
): number {
  const [r1, g1, b1] = oklchToSrgb(l1, c1, h1);
  const [r2, g2, b2] = oklchToSrgb(l2, c2, h2);
  const lum1 = relativeLuminance(r1, g1, b1);
  const lum2 = relativeLuminance(r2, g2, b2);
  const lighter = Math.max(lum1, lum2);
  const darker  = Math.min(lum1, lum2);
  return (lighter + 0.05) / (darker + 0.05);
}

/**
 * Find the lightness value that achieves a target contrast ratio
 * against a reference color by binary search.
 */
function findLightnessForContrast(
  referenceL: number, referenceC: number, referenceH: number,
  targetRatio: number, direction: 'lighter' | 'darker' = 'lighter'
): number {
  let lo = direction === 'lighter' ? referenceL : 0;
  let hi = direction === 'lighter' ? 100 : referenceL;

  for (let i = 0; i < 30; i++) {
    const mid = (lo + hi) / 2;
    const ratio = contrastOklch(
      referenceL, referenceC, referenceH,
      mid, referenceC, referenceH
    );

    if (direction === 'lighter') {
      if (ratio < targetRatio) lo = mid;
      else hi = mid;
    } else {
      if (ratio < targetRatio) hi = mid;
      else lo = mid;
    }
  }

  return (lo + hi) / 2;
}

/**
 * Verify WCAG compliance level.
 */
type WcagLevel = 'FAIL' | 'AA' | 'AAA';
type TextSize = 'normal' | 'large';

function wcagLevel(ratio: number, size: TextSize = 'normal'): WcagLevel {
  if (size === 'large') {
    if (ratio >= 4.5) return 'AAA';
    if (ratio >= 3.0) return 'AA';
  } else {
    if (ratio >= 7.0) return 'AAA';
    if (ratio >= 4.5) return 'AA';
  }
  return 'FAIL';
}
```

// ── Oklab / OKLCH Core Conversions ─────────────────────────────────────────

function oklchToOklab(L: number, C: number, h: number): [number, number, number] {
  const hRad = (h * Math.PI) / 180;
  const a = C * Math.cos(hRad);
  const b = C * Math.sin(hRad);
  return [L / 100, a, b]; // L is 0-100 in OKLCH, 0-1 in Oklab
}

function oklabToLinearSrgb(L: number, a: number, b: number): [number, number, number] {
  const l_ = L + 0.3963377774 * a + 0.2158037573 * b;
  const m_ = L - 0.1055613458 * a - 0.0638541728 * b;
  const s_ = L - 0.0894841775 * a - 1.2914855480 * b;

  const l = l_ * l_ * l_;
  const m = m_ * m_ * m_;
  const s = s_ * s_ * s_;

  return [
    +4.0767416621 * l - 3.3077115913 * m + 0.2309699292 * s,
    -1.2684380046 * l + 2.6097574011 * m - 0.3413193965 * s,
    -0.0041960863 * l - 0.7034186147 * m + 1.7076147010 * s,
  ];
}

function linearToGamma(c: number): number {
  if (c <= 0.0031308) return 12.92 * c;
  return 1.055 * Math.pow(c, 1 / 2.4) - 0.055;
}

function oklchToSrgb(L: number, C: number, h: number): [number, number, number] {
  const [labL, labA, labB] = oklchToOklab(L, C, h);
  const [lr, lg, lb] = oklabToLinearSrgb(labL, labA, labB);
  return [
    Math.round(Math.max(0, Math.min(1, linearToGamma(lr))) * 255),
    Math.round(Math.max(0, Math.min(1, linearToGamma(lg))) * 255),
    Math.round(Math.max(0, Math.min(1, linearToGamma(lb))) * 255),
  ];
}

/**
 * Generate an 11-stop color scale (50-950) from a hue and chroma.
 * Lightness follows a perceptual curve; chroma is tapered at extremes.
 */
function generateScale(config: ScaleConfig): Record<ScaleStop, string> {
  const { hue, chroma, chromaCurve = 0.85 } = config;
  const result: Record<string, string> = {};

  for (const stop of SCALE_STOPS) {
    const L = SCALE_LIGHTNESS[stop];
    // Reduce chroma at lightness extremes to stay in gamut
    const distFromCenter = Math.abs(L - 50) / 50; // 0 at 50, 1 at 0/100
    const chromaMultiplier = 1 - distFromCenter * chromaCurve;
    const C = chroma * chromaMultiplier;

    result[stop] = `oklch(${L}% ${C.toFixed(4)} ${hue})`;
  }

  return result as Record<ScaleStop, string>;
}

/**
 * Derive dark-mode equivalents from light-mode scale values.
 * Strategy: Invert the lightness scale (97→8, 93→16, ...) while
 * slightly reducing chroma (colors appear more vivid in dark mode).
 * Adjust hue by +5° to compensate for Hunt effect.
 */
function deriveDarkScale(lightScale: Record<ScaleStop, string>): Record<ScaleStop, string> {
  const darkScale: Record<string, string> = {};

  // Material Design 3 pattern: map light→dark lightness
  const darkLightness: Record<ScaleStop, number> = {
    50:  8,    // darkest (was lightest)
    100: 12,
    200: 16,
    300: 22,
    400: 28,
    500: 38,   // middle becomes slightly darker
    600: 48,
    700: 60,
    800: 74,
    900: 88,
    950: 96,   // lightest (was darkest)
  };

  for (const [stopStr, value] of Object.entries(lightScale)) {
    const stop = Number(stopStr) as ScaleStop;
    // Parse the OKLCH string: oklch(L% C H)
    const match = value.match(/oklch\((\d+(?:\.\d+)?)%\s+([\d.]+)\s+([\d.]+)\)/);
    if (!match) { darkScale[stopStr] = value; continue; }

    const chroma = parseFloat(match[2]);
    const hue = parseFloat(match[3]);
    const newL = darkLightness[stop];

    // Reduce chroma by ~20% and shift hue by +5° for perceptual neutrality
    const newC = chroma * 0.80;
    const newH = (hue + 5) % 360;

    darkScale[stopStr] = `oklch(${newL}% ${newC.toFixed(4)} ${newH.toFixed(1)})`;
  }

  return darkScale as Record<ScaleStop, string>;
}

/**
 * Flatten a nested tokens object into CSS custom property declarations.
 */
function toCssCustomProperties(
  tokens: Record<string, string | Record<string, string>>,
  prefix: string = '--color'
): string {
  const lines: string[] = [];

  function walk(obj: Record<string, unknown>, path: string[]) {
    for (const [key, value] of Object.entries(obj)) {
      if (typeof value === 'string') {
        lines.push(`  ${prefix}-${[...path, key].join('-')}: ${value};`);
      } else if (typeof value === 'object' && value !== null) {
        walk(value as Record<string, unknown>, [...path, key]);
      }
    }
  }

  walk(tokens, []);
  return lines.join('\n');
}

/**
 * Generate the complete CSS custom properties output for light and dark modes.
 */
function generateFullCss(config: PaletteConfig = defaultConfig): string {
  const palettes = {
    primary:   generateScale(config.primary),
    secondary: generateScale(config.secondary),
    accent:    generateScale(config.accent),
    neutral:   generateScale(config.neutral),
  };

  const darkPalettes = {
    primary:   deriveDarkScale(palettes.primary),
    secondary: deriveDarkScale(palettes.secondary),
    accent:    deriveDarkScale(palettes.accent),
    neutral:   deriveDarkScale(palettes.neutral),
  };

  const semantic: Record<string, SemanticTokens> = {};
  const darkSemantic: Record<string, SemanticTokens> = {};

  for (const [name, cfg] of Object.entries(semanticConfigs)) {
    semantic[name] = deriveSemanticTokens(cfg);
    darkSemantic[name] = deriveDarkSemanticTokens(semantic[name], cfg);
  }

  // Build the nested token structure
  const lightTokens: Record<string, unknown> = {};
  for (const [paletteName, scale] of Object.entries(palettes)) {
    lightTokens[paletteName] = scale;
  }
  for (const [semName, tokens] of Object.entries(semantic)) {
    lightTokens[semName] = tokens;
  }

  const darkTokens: Record<string, unknown> = {};
  for (const [paletteName, scale] of Object.entries(darkPalettes)) {
    darkTokens[paletteName] = scale;
  }
  for (const [semName, tokens] of Object.entries(darkSemantic)) {
    darkTokens[semName] = tokens;
  }

  const lightCss = toCssCustomProperties(lightTokens);
  const darkCss = toCssCustomProperties(darkTokens);

  return [
    '/* ══════════════════════════════════════════════════════════════════════',

}

## 6. Dark Mode Automatic Derivation

### Derivation Strategy

The dark mode derivation applies three transformations:

1. **Lightness Inversion:** Maps the 11-stop scale (97→8, 93→12, ..., 8→96) — this is NOT a simple mirror but a perceptually-calibrated mapping based on Material Design 3's tonal palette research.

2. **Chroma Reduction (×0.80):** Colors appear ~20% more vivid in dark environments (the Hunt effect). Reducing chroma compensates so perceived saturation remains consistent.

3. **Hue Shift (+5°):** A small counter-clockwise rotation compensates for the Bezold–Brücke effect, where hues appear to shift at different luminance levels.

### Light → Dark Mapping Table

| Stop | Light L | Dark L | Purpose |
|------|---------|--------|---------|
| 50   | 97%     | 8%     | Page background |
| 100  | 93%     | 12%    | Subtle backgrounds |
| 200  | 85%     | 16%    | Hover states |
| 300  | 74%     | 22%    | Disabled states |
| 400  | 62%     | 28%    | Low-emphasis text |
| 500  | 50%     | 38%    | Base color, icons |
| 600  | 41%     | 48%    | Emphasis |
| 700  | 32%     | 60%    | High-emphasis text |
| 800  | 24%     | 74%    | Text on dark |
| 900  | 16%     | 88%    | Strong emphasis |
| 950  | 8%      | 96%    | Inverted surface |

### Implementation Pattern

```css
/* Theme switching via data attribute */
[data-theme="dark"] {
  color-scheme: dark;
}

/* Or via class-based toggling */
<html class="dark">

/* Or via system preference with explicit override */
@media (prefers-color-scheme: dark) {
  :root:not([data-theme="light"]) { /* dark vars */ }
}
```

The `:root:not([data-theme="light"])` pattern ensures system-preference dark mode activates automatically but yields to an explicit `data-theme="light"` override — the user's explicit choice always wins.

## 7. Color Scale Generation (50–950)

### Scale Architecture

The 11-stop scale (50, 100, 200, 300, 400, 500, 600, 700, 800, 900, 950) follows Material Design 3 conventions:

```
Stop:  50    100   200   300   400   500   600   700   800   900   950
L%:    97    93    85    74    62    50    41    32    24    16    8
       ↑                                                            ↑
    Lightest                                                     Darkest
    (page bg)                                                  (deep ink)
```

### Chroma Taper Curve

At lightness extremes (near 0% or 100%), high-chroma colors clip outside the sRGB gamut. The generator applies a cosine-like taper:

```
C_effective(stop) = C_base × (1 − |L(stop) − 50|/50 × chromaCurve)
```

Where `chromaCurve` (default 0.85) controls how aggressively chroma falls off:

- **0.0:** No taper — may produce gamut-clipped colors at extremes
- **0.85:** Smooth taper — perceptually pleasing, gamut-safe (default)
- **1.0:** Full taper — chroma reaches 0 at L=0% and L=100%

### Generated Scale Example: Primary (h=264, c=0.18)

```
 50: oklch(97% 0.027 264)   ← near-white violet tint
100: oklch(93% 0.049 264)   ← light violet
200: oklch(85% 0.073 264)   ← soft violet
300: oklch(74% 0.107 264)   ← muted violet
400: oklch(62% 0.143 264)   ← medium violet
500: oklch(50% 0.180 264)   ← vivid violet (base)
600: oklch(41% 0.152 264)   ← deep violet
700: oklch(32% 0.125 264)   ← dark violet
800: oklch(24% 0.100 264)   ← very dark violet
900: oklch(16% 0.076 264)   ← near-black violet
950: oklch(8%  0.051 264)   ← ink violet
```

## Appendix B: Design Decisions & Rationale

### Why OKLCH over HSL?

| Property | HSL | OKLCH |
|----------|-----|-------|
| Perceptual uniformity | Poor — L=50% yellow looks much brighter than L=50% blue | Excellent — equal L = equal perceived lightness |
| Hue linearity | Blues shift to purple at different S/L | Hues are linearly spaced and stable |
| Gamut handling | No clipping information | Chroma maps directly to sRGB/P3 gamut |
| Dark mode generation | Requires manual overrides | Invert L, reduce C by ~20%, shift H by +5° |
| Browser support | 100% | ~93% (Chrome 111+, Firefox 113+, Safari 15.4+) |

### Why 11-Stop Scale Instead of 10?

The 50/950 extensions match Material Design 3's tonal palette system:

- **50:** Near-white for page backgrounds and subtle tints
- **950:** Near-black for deep ink, shadows, and maximum contrast
- Standard 100-900 covers typical UI needs; 50 and 950 provide the extremes

### Why Hue Shift in Dark Mode?

The Bezold–Brücke effect causes perceived hue shifts at different luminance levels. A +5° shift in dark mode compensates so the color "reads" the same to users. This is optional but improves the subjective quality of dark-mode palettes.

### Fallback Strategy

For legacy browser support, use `@supports`:

```css
.my-element {
  /* Fallback: closest sRGB approximation */
  background: #6b5ce7;
  /* OKLCH: progressively enhanced */
  background: oklch(50% 0.18 264);
}

/* Or use @supports */
@supports (color: oklch(0% 0 0)) {
  .my-element {
    background: oklch(50% 0.18 264);
  }
}
```
