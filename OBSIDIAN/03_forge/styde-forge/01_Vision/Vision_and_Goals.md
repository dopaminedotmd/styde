# Vision and Goals

**Styde Forge v3.0 — The Crucible**

---

## 1. Core Vision

> A portable evolutionary elite-agent refinery — where raw blueprints are
> transformed into world-class specialized agents through precision and
> iterative purity.

Styde Forge v3.0 is not a content factory. It does not maximize GB/hour.
It maximizes **agent quality per generation** through continuous evaluation,
teacher-guided improvement, and Caveman Ultra efficiency (70% fewer tokens).

---

## 2. Primary Goals

| # | Goal | Success Metric |
|---|------|----------------|
| 1 | **Quality over quantity** | Every agent ≥ 80/100 on eval |
| 2 | **Full portability** | Works on Machine-A and Machine-B without manual config |
| 3 | **One-prompt import** | Import entire forge on any machine with one prompt |
| 4 | **Traceable evolution** | Complete lineage from v1.0 to current for every agent |
| 5 | **Self-improving** | Each generation measurably better than the last |
| 6 | **Hardware-adaptive** | Auto-detects and optimizes for available resources |

---

## 3. Non-Goals (Explicitly Excluded in v3.0)

- Real-time multi-user collaboration
- Web dashboard / GUI
- Cloud orchestration
- Reinforcement learning from human feedback (RLHF) integration
- Full ML weight training (agents are text/code-based)

---

## 4. Content to Generate

| Type | Description | Size Target |
|------|-------------|-------------|
| Specialized Agents | World-class, domain-specific, fully evaluated | 25 GB (250-350 agents) |
| Knowledge Bases | Indexed, sourced, non-redundant domain knowledge | 8 GB (8-10 domains) |
| Skills & Hooks | Modular, reusable, tested building blocks | 5 GB |
| Evaluation Results | Complete history with lineage | 4 GB |
| Checkpoints | Rolling safety snapshots | 3 GB |
| Meta-data & Indexes | Manifest, teacher logs, health data | 3 GB |

---

## 5. Six Core Domains

1. **Coding & Software Engineering** — Code generation, review, architecture
2. **Research Synthesis & Knowledge** — Paper analysis, literature review
3. **Automation & Orchestration** — Workflow automation, system integration
4. **Documentation & Communication** — Technical writing, clarity
5. **Testing & Evaluation** — Test generation, quality assurance
6. **Meta-Self-Improvement** — Improving the forge itself

---

## 6. Quality Standards

| Level | Score | Action |
|-------|-------|--------|
| Elite | ≥ 90/100 | Saved with full documentation + prioritized |
| World-Class | ≥ 85/100 | Saved. Used as teacher/coach candidate |
| Production | ≥ 80/100 | Saved to USB |
| Needs Work | 70-79 | Iterate (max 3 attempts) |
| Rejected | < 70 | Discarded. Lessons saved to Historical Learning |

---

## 7. Long-Term Vision

After 8-12 weeks of continuous operation:
- 250-350 elite agents across 6 domains
- Each agent is traceable, evaluable, and built with optimal model for its purpose
- Historical Learning knows which patterns, models, and prompts work best
- The USB is a self-contained, importable knowledge reactor
